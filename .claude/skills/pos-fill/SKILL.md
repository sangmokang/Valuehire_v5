# 포지션 등록 모달 채우기 (잡코리아 / 사람인) — 한 후보자용

> 사장님이 후보자 이력서를 보다가 [포지션 제안](잡코리아) 또는 [이직 제안](사람인)을 눌러 "포지션 등록" 모달을 띄우고 "**이거 뭐라고 등록해?**" 라고 물으실 때를 위한 스킬. 후보자 1명에 맞는 포지션을 골라 모달 필드를 paste-ready 로 채워 드린다.
>
> 자매 스킬: 인재 검색~제안 전체 한 턴은 `jobkorea-talent-sourcing` / `saramin-talent-sourcing`. 본 스킬은 그 흐름의 **"포지션 등록 모달 한 칸 채우기"** 부분만 빠르게 떼어낸 것.

## 0. 트리거 (사장님이 이렇게 말하면 발동)

- "이거 등록을 뭐라고 해야해" / "이거 뭐라고 등록해"
- "포지션 등록 뭐라고" / "포지션 등록 채워줘"
- "이 후보 포지션 등록" / "이 사람한테 뭐 제안하지"
- (스크린샷으로 포지션 등록 모달을 보여주시면서) 위 질문

## 1. 절대 규칙

| # | 규칙 |
|---|------|
| F0 | **발송 금지.** "포지션 등록" 버튼까지만 값을 채워 드린다. [제안 발송]/[포지션 제안 발송]은 라이선스 차감 + 후보자에게 실제 전달 = 사장님 직접. |
| F1 | **지금 보고 있는 후보자에 맞춰라.** 후보자의 보유스킬·총경력·희망직무·학력을 읽고 가장 가까운 포지션을 고른다. 안 맞으면 솔직히 말한다(억지 매칭 금지). |
| F2 | **포지션 후보군 = 현재 캠페인.** 기본은 사장님이 진행 중인 캠페인 포지션 풀. 불명확하면 §3 소스에서 찾고, 그래도 없으면 1줄로 묻는다. |
| F3 | **추정 금지.** 회사·JD 숫자는 §3 캐시/검색으로 확인된 것만. 모르면 비워두거나 "면접 후 결정". |
| F4 | **120일 자동삭제 안내.** 잡코리아: "등록 후 120일간 발송 이력 없으면 자동 삭제" — 등록만 하고 안 보내면 사라짐을 사장님께 알린다. |

## 2. 잡코리아 "포지션 등록" 모달 필드 맵

(이력서 보기 → 우측 [포지션 제안] → [포지션 등록] 모달)

| 필드 | 필수 | 채우는 값 | 기본값 |
|------|:---:|----------|--------|
| **채용 포지션** | ★ | 포지션 풀네임 (예: `AX Backend Engineer (NestJS·AI 에이전트)`) | — |
| **고용형태** (드롭다운) | | 정규직 / 계약직 / … | **정규직** |
| **직무** (직무선택) | ★ | 잡코리아 직무 카테고리 1개 (예: 서버/백엔드개발, AI/ML엔지니어, 마케팅) | 포지션에 맞게 |
| **근무지 주소** | | 회사 본사 주소 (예: 서울 서초구) | 회사 위치 |
| 근무부서 / 직급 / 직책 | | 보통 비움(무관) | 비움 |
| 최소연봉~최대연봉 | | 비우고 **"면접 후 결정" 체크** | 면접 후 결정 ✅ |
| **입사후 업무** | ★ | JD 주요업무 글머리표 그대로 | — |
| **우대사항** | | JD 우대사항 글머리표 그대로 | — |

## 3. 사람인 "포지션 등록(이직 제안)" 모달 필드 맵

(saramin-talent-sourcing §10 참조 — 안정 selector)

| 필드 | name 속성 | 비고 |
|------|----------|------|
| 회사명 | `offerCompanyNm` | 채용 회사 |
| 포지션명 | `input[placeholder="포지션명 입력"]` | [+포지션 추가] 후 |
| 제안 내용 ① | `jobOffer.offerComment` | 회사 브리핑(§17) + 인사 (한도 2,000자) |
| 업무 내용 ② | `jobOffer.chargeWork` | JD 주요업무·자격·우대 (한도 2,000자) |

## 4. 포지션 데이터 소스 (우선순위)

1. **포지션 캐시** — `~/.cache/saramin-positions/unified.jsonl`, `~/.cache/jobkorea-positions/unified.jsonl` (회사·포지션명)
2. **회사 조사 캐시** — `~/.cache/saramin-company-research/<slug>.json` (회사 브리핑용)
3. **ClickUp FY26ClientsPosition** 보드 task `jd_text`
4. **회사 공식 채용 페이지** — 뤼튼 `wrtn.career.greetinghr.com`, 핵클 `careers.hackle.io` 등 (career.wrtn.io 는 WebFetch 403 → WebSearch 로 우회)

## 5. 실행 절차

1. **후보자 식별** — 사장님 화면(스크린샷) 또는 Chrome 탭 URL의 `rNo=`/`resume/view` 에서 이력서 읽기: 보유스킬·총경력·희망직무·학력.
2. **포지션 매칭** — §4 소스의 캠페인 포지션 중 후보자에 가장 맞는 1개 선택. (엔지니어→AX/엔지니어 직무, 마케터→Growth/Brand Marketer 등). 적합도 애매하면 1·2순위 둘 다 제시.
3. **필드값 생성** — 해당 플랫폼(§2 잡코리아 / §3 사람인) 필드맵에 맞춰 paste-ready 표로 출력. JD 주요업무·우대사항은 §4 에서 확인된 원문 기준.
4. **적합도 1줄** — 후보자 ↔ 포지션 매칭 근거/한계 솔직히.
5. **안내** — F0(발송은 직접), F4(120일 자동삭제).

```
🟢 [회사] [포지션] 으로 등록 추천 — 후보자: AOO (AI/ML 5년차)

채용 포지션 : <값>          (복사)
고용형태    : 정규직
직무        : <카테고리>
근무지      : <주소>
연봉        : 면접 후 결정 ✅
입사후 업무 :
<글머리표>
우대사항    :
<글머리표>

· 적합도: <한 줄 근거/한계>
· 발송(포지션 제안)은 사장님이 직접 — 등록 후 120일 미발송 시 자동 삭제
```
