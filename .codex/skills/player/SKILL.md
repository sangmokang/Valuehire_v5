# player — 국가대표/프로팀 선수단 조사 스킬

> Codex CLI에서는 `~/.codex/skills/player/SKILL.md`(이 파일과 동일 내용)로 동일하게 `$player`로 호출한다. 두 파일 중 하나를 고치면 반드시 다른 쪽도 동기화한다.

## 0. 언제 쓰나
사용자가 `/player <팀명 또는 선수명>` 형태로 호출하거나, "이 국가대표/이 클럽 선수들 조사해줘" 류의 요청을 할 때. 팀 전체(스쿼드)든 선수 1명이든 모두 이 스킬로 처리한다.

## 1. 입력 파싱
- `/player 스페인 국가대표` `/player 맨체스터시티` 처럼 **팀명**이면 → 최신 공식 명단(국가대표는 최근/예정 메이저대회 최종엔트리, 클럽은 현재 1군 스쿼드) 전원을 대상으로 한다.
- `/player 손흥민` 처럼 **선수명 1명**이면 → 그 선수 1명만 동일 포맷으로 조사한다.
- 사용자가 특정 대회/시즌을 명시하면(예: "2026 월드컵", "이번 시즌") 그 기준으로 "최근 대회/시즌 스탯" 항목을 채운다. 명시가 없으면 가장 최근에 끝났거나 진행 중인 메이저 대회/시즌을 기준으로 삼는다.

## 2. 명단 확보
WebSearch로 공식 최종 명단부터 확정한다(예: "{팀명} 최종 명단", "{팀명} squad official list"). 포지션별 인원(GK/DF/MF/FW)을 먼저 표로 파악한 뒤 3단계로 넘어간다. 명단이 불확실하면(이적시장 도중이거나 소식이 엇갈리면) 사용자에게 먼저 확인한다.

## 3. 고정 필드 포맷 (절대 순서·항목 변경 금지)
선수마다 아래 11개 항목을 **이 순서 그대로** 채운다. 확인 안 되는 항목은 절대 추측하지 말고 `공개 정보 없음`이라고 쓴다. 특히 "연습 루틴" "존경하는 선수" "100m 스프린트 속도" "평균 활동 거리"는 공개 인터뷰/공식 트래킹 데이터가 없는 선수가 대다수이므로, 검색으로 확인된 것만 쓴다.

```
## [선수명]
- 나이 / 포지션 / 주로 뛰는 포메이션(팀 기준)
- 커리어: 현 소속팀 / 과거 소속팀(유스 포함, 이적 흐름)
- 연령별 국가대표: U-17/U-19/U-21/U-23 등 경력
- 고향
- 축구 기량이 비약적으로 성장한 시기와 그 계기/방법 (구체적 시즌·사건 기반)
- 연습 루틴
- 주특기
- 존경하는 선수
- [최근 대회/시즌] 스탯: 출전 수, 선발/교체, 골/어시스트 등 실제 수치
- 평균 활동 거리(km/경기)
- 100m 스프린트 속도
```

## 4. 병렬 리서치 (스쿼드 전체일 때)
- 인원이 많으면(대략 15명 이상) 포지션 묶음으로 3~4그룹(GK+DF / MF / FW, 또는 인원에 맞춰 균등 분배)으로 나눠 **Agent 툴을 그룹당 1개씩 동시에(같은 메시지에서 병렬) 호출**한다. 선수 1명이면 그룹화 없이 Agent 1개(또는 직접 WebSearch)로 처리해도 된다.
- 각 Agent 프롬프트에는 반드시: 대상 선수 목록(이름+소속팀), §3 포맷 원문 그대로, "날조 금지 — 확인 안 되면 공개 정보 없음", 그리고 아래 **출력 안정성 문구**를 포함한다.

### 출력 안정성 문구 (필수 — 그대로 붙여넣기)
> CRITICAL OUTPUT REQUIREMENT: Your FINAL message (the very last text you output before stopping) MUST contain the ENTIRE markdown text for all players, written out in full, verbatim. Do not write "done" or "task complete" or any meta-commentary as your final message — the final message body itself IS the deliverable and will be parsed programmatically. If your final message doesn't contain the full markdown text, the task has failed.

**알려진 실패 모드**: 위 문구를 넣어도 서브에이전트가 "Task complete", "Done." 같은 메타 문구만 반환하고 실제 마크다운을 누락하는 경우가 있었다(2026-07-15 스페인 26인 조사에서 3개 그룹 모두 재현). SendMessage로 재요청해도 같은 증상이 반복될 수 있다. 이 경우:
1. 같은 프롬프트로 **완전히 새 Agent를 재호출**한다(resume 아님, 신규 스폰).
2. 그래도 최종 텍스트가 비어 있으면, 에이전트 트랜스크립트에서 직접 회수한다: 반환된 `agentId`로 심볼릭 링크 `/private/tmp/claude-501/.../tasks/<agentId>.output` → 실제 파일 `~/.claude/projects/<project>/<session>/subagents/agent-<agentId>.jsonl`. 이 JSONL을 파이썬으로 파싱해 `type=="assistant"`인 메시지들의 `content[].text` 중 **가장 긴 텍스트**를 골라내면 실제 산출물이다(짧은 "done"류 메시지는 자동으로 걸러진다).

## 5. 조립
그룹별 결과를 받으면(각 결과 텍스트에서 "Sources:" 이하 각주 섹션은 잘라내고) `## [선수명]` 단위로 이어붙여 하나의 마크다운으로 합친다. 맨 위에 `## 개요` 섹션을 추가해 팀명·기준 대회/시즌·명단 발표일·이번 조사에서 "공개 정보 없음" 처리 기준을 1~2문장으로 밝힌다.

## 6. 포스팅 (§0.6 콘텐츠 라우팅 SOT 그대로 따름)
축구 콘텐츠이므로 claude.ai Artifact가 아니라 **로컬 Football 앱**에 실제로 포스팅한다.
1. `curl -s http://127.0.0.1:5178/api/posts`로 서버 확인, 응답 없으면 `cd apps/Football && ./run.sh &`로 기동.
2. `POST http://127.0.0.1:5178/api/posts`로 등록. category는 스쿼드 전체 조사면 `지식 체계`(참고자료 성격), 선수 1명 단건 프로필이면 `축구 포스팅` 중 상황에 맞게 선택. `title`은 "{팀명} {대회/시즌} 선수단 프로필" 형식, `themes`는 `#{팀명} #{대회/시즌} #선수프로필` 식 해시태그, `body`는 5단계에서 조립한 마크다운 전체.
3. **라이브 증거로 검증**(완료 주장 전 필수): `curl -s http://127.0.0.1:5178/api/posts`로 방금 등록한 `id`가 목록에 뜨는지 확인.
4. 축구가 아닌 다른 종목(야구·농구 등)을 요청받으면 apps/Football 라우팅 대상이 아니므로, 포스팅 전에 사용자에게 어디에 올릴지 먼저 확인한다(전역 CLAUDE.md §0.6은 축구/재즈만 다룸).

## 7. 사용자 보고
전역 CLAUDE.md §0.7에 따라 **존댓말**로 요약 보고한다: 조사 인원, 포스팅된 Football 앱 id/제목, "공개 정보 없음"으로 남은 항목이 많은 경우 그 사실을 솔직히 알린다(과장 금지).
