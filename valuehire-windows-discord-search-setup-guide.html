Valuehire Windows 멤버용 Discord AI Search 환경 구축 가이드

레포지토리와 개발 환경이 없는 윈도우 멤버도 Discord 개인 DM에서 ClickUp 링크, Wanted(원티드) 링크, 채용공고 본문 복붙으로 후보자 Search를 요청할 수 있게 만드는 운영 가이드입니다.

대상: Windows 멤버방식: Discord 개인 DM원칙: fail-closed레포 설치 불필요

핵심 요약

멤버 PC에는 Valuehire_v5 레포가 없어도 됩니다. 멤버는 Discord만 설치하고, 운영자는 Hermes Gateway와 Search 도구를 중앙 PC 또는 서버에서 실행합니다.

멤버가 Discord 개인 DM으로 Search + 포지션을 보내면, Hermes가 포지션을 읽고 AI Search를 수행합니다. 단, 허용된 Discord ID의 개인 DM만 실행됩니다.

1. 전체 구조

멤버 Windows PC

  • Discord 데스크톱 앱 또는 웹 브라우저
  • ClickUp/Wanted 링크 복사
  • 채용공고 본문 복붙
  • 개발 레포, Python, Node 설치 불필요

Discord

  • Hermes 봇과 개인 DM
  • 허용 Discord ID만 실행
  • 서버 채널에서는 후보 검색 실행 금지
  • 포지션 누락 시 되묻기

운영자 Hermes 환경

  • Hermes Gateway 실행
  • AI Search 도구 연결
  • ClickUp Activity 결과 포맷 관리
  • 사람인·잡코리아·LinkedIn 게이트 관리
Windows 멤버 → Discord 개인 DM → Hermes Gateway → Valuehire AI Search → Discord 답변 / ClickUp Activity 준비

2. Windows 멤버가 준비할 것

2-1. Discord 설치 또는 접속

둘 중 하나만 쓰면 됩니다.

  • 데스크톱 앱: https://discord.com/download
  • 웹: https://discord.com/app

2-2. 운영자에게 Discord ID 전달

개인 DM 허용 목록에 들어가야 Search가 실행됩니다. Discord ID는 아래 순서로 복사합니다.

  1. Discord 설정 → 고급 → 개발자 모드 켜기
  2. 왼쪽 아래 본인 프로필 우클릭
  3. 사용자 ID 복사
  4. 운영자에게 전달

허용 목록 소스는 운영자 레포의 docs/search-access.md입니다. 멤버는 이 파일을 직접 수정하지 않습니다.

2-3. Hermes 봇에게 개인 DM 보내기

서버 채널이 아니라 봇과의 개인 DM에서만 Search가 실행됩니다.

3. 멤버가 Search 요청하는 방법

아래 3가지 입력을 지원합니다. 공통 규칙은 Search 의도 + 포지션입니다.

입력 방식예시처리
ClickUp 링크 AI Search https://app.clickup.com/t/86abc123 ClickUp 포지션을 읽어 Search
Wanted 링크 서치 https://www.wanted.co.kr/wd/123456 Wanted 공고 URL을 포지션으로 사용
본문 복붙 Search
회사소개: ...
주요업무: ...
자격요건: ...		 복붙한 JD를 포지션으로 사용
중요: Search만 보내고 포지션이 없으면 후보 검색을 시작하지 않습니다. Hermes가 “ClickUp 링크, Wanted 링크, 또는 채용공고 본문을 보내주세요”라고 되묻는 것이 정상입니다.

4. 정상 동작 기준

  • 허용: 허용 목록에 있는 멤버가 Hermes 봇에게 개인 DM으로 포지션 포함 Search 요청
  • 차단: 서버 채널 메시지, 허용되지 않은 Discord ID, 포지션 없는 Search 요청
  • 보류: ClickUp Activity 결과에 필수 4개 필드가 없으면 쓰기 보류

ClickUp Activity 필수 4개 필드

  1. Profile URL
  2. 점수
  3. 왜 잘 맞는지
  4. 후보자 프로필 요약

5. 운영자가 준비할 것

아래 작업은 중앙 운영 PC 또는 서버에서만 필요합니다. Windows 멤버 PC에서 실행하지 않습니다.

5-1. Hermes Gateway 상태 확인

hermes gateway status

5-2. Discord 토큰과 허용 사용자 확인

Hermes 환경 파일에는 Discord 봇 토큰과 허용 사용자 설정이 필요합니다. 값은 절대 문서나 채팅에 붙이지 않습니다.

# 예시: 값은 실제로 공유하지 마세요.
DISCORD_BOT_TOKEN=[REDACTED]
DISCORD_ALLOWED_USERS=[REDACTED]

Hermes 환경 파일과 프로젝트 .env.local은 서로 다릅니다. Hermes Gateway가 읽는 값은 보통 ~/.hermes/.env입니다.

5-3. Search 접근 문서 업데이트

멤버 Discord ID를 docs/search-access.md의 Discord Contacts 표에 추가합니다. 15~22자리 숫자 Discord ID만 허용됩니다.

5-4. 변경 반영

환경변수나 gateway 설정을 바꿨다면 Hermes Gateway 재시작 또는 Discord에서 /reload가 필요할 수 있습니다. 재시작은 실행 중 서비스에 영향을 줄 수 있으므로 운영자가 승인 후 진행합니다.

6. 검증 명령

운영자 레포에서 아래 명령으로 라우팅, 필수 필드, dry-run 산출물을 검증합니다.

cd /Users/kangsangmo/Desktop/Valuehire_v5
python3 -m unittest tests/test_multi_position_sourcing.py -v
python3 -m tools.multi_position_sourcing.dry_run --output artifacts/multi_position_sourcing/dry-run-latest.json

성공 기준:

  • 단위 테스트 전체 통과
  • dry-run 산출물의 side_effects가 모두 false
  • discord_dm_routing.allowed가 허용 사용자 개인 DM에서만 true
  • ClickUp Activity 샘플에 필수 4개 필드 포함

9. 문제 해결

증상원인해결
Search가 실행되지 않음 개인 DM이 아니거나 허용 Discord ID가 아님 DM 여부와 docs/search-access.md 등록 여부 확인
포지션을 다시 물어봄 Search만 보내고 포지션이 없음 ClickUp 링크, Wanted 링크, 또는 JD 본문을 함께 전송
새 멤버 등록 후에도 차단됨 Gateway가 변경 전 환경을 읽고 있음 운영자 승인 후 gateway reload 또는 restart
ClickUp Activity 쓰기가 보류됨 필수 4개 필드 중 하나 누락 Profile URL, 점수, 적합 이유, 후보자 요약 포함 여부 확인
라이브 Search가 닫힘 OWNER_SIGNOFF 또는 AI_SEARCH_MULTIENGINE 게이트가 닫힘 dry-run 결과를 먼저 보고, 사장님 승인 후 게이트 열기
