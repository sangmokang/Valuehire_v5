## 1. 한 줄 요약

> **"/url 은 밥상 차리기, /humansearch 는 밥 먹기, /aisearch 는 장보기부터 풀코스."**

| 스킬 | 역할 (사람 말) | 하는 일 | 절대 안 하는 일 |
|---|---|---|---|
| **/url** | **준비** — 링크드인 검색을 미리 걸어둔다 | RPS 프로젝트 확인/생성 → JD에서 검색식(Boolean 3단) 뽑아 실행 → 라이브 검색 링크 1개 + 레시피를 ClickUp 포지션 댓글에 저장 | 후보를 열어보거나 채점·발송하지 않음 |
| **/humansearch** | **수확** — 걸어둔 검색을 돌며 후보를 골라낸다 | 미리 걸린 검색결과(RPS·사람인·잡코리아) 순회 → 채점(학력30·직무50·논리10·이직안정10, 합격 70+) → ClickUp FY26AI_Search 등록 + 저장 + DM 보고. 요청 시 1명 개인화 제안문까지 | 검색어 생성·필터 입력 안 함. **발송(Send) 절대 자동 안 누름** |
| **/aisearch** | **풀코스** — JD부터 후보 리스트까지 처음부터 끝까지 | JD 분석 → 키워드 5축 → **사람인·잡코리아·LinkedIn RPS 병렬 검색** → 적합도 평가 → 표준 출력계약(profile_url·score·why_fit·summary) → 보고 | 임의 검색 시작 안 함(포지션 입력 필수). 발송 자동 금지 |

핵심 경계: **검색을 "만드는" 건 /url(링크드인)과 /aisearch(3사), 검색을 "소비하는" 건 /humansearch.**

## 2. 표준 연계 흐름 (권장)

### 흐름 A — 링크드인 중심 · 가장 간소 (일상 기본)

```
포지션 등록/갱신
   │
   ▼
① /url            ← 인자 없이 치면 살아있는 포지션 전수를 미리 세팅 (배치성 작업, 밤에 돌려도 됨)
   │  산출물: ClickUp 포지션 댓글에 [라이브 검색 링크 1개 + Boolean 레시피]
   ▼
② /humansearch 코드잇 백엔드     ← 후보가 필요한 순간, 포지션 이름만 부른다
   │  산출물: FY26AI_Search 부모 Task + 후보 Subtask + DM 보고 (+요청 시 개인화 제안문 1건)
   ▼
③ 사장님이 직접 발송 (Send 는 항상 사람 손)
```

- 왜 이게 최선: /url 이 **비싼 준비(검색식 설계·RPS 세팅)** 를 미리 끝내두므로, 후보가 필요한 순간에는 **한 마디**로 수확만 하면 된다. 두 턴이 완전히 분리돼 사장님 크롬 점유·캡차에 강하다.
- 조건: 포지션이 ClickUp 리스트 901814621569 에 있어야 한다.

### 흐름 B — 3사 전방위가 필요할 때 (/aisearch)

```
/aisearch <포지션>  (+"잡코리아도 같이" / "링크드인도")
   → 키워드 전략부터 사람인·잡코리아·RPS 병렬 검색 → 평가 → 후보 리스트
```

- 쓰는 경우: ① **사람인·잡코리아 채널까지** 한 번에 봐야 할 때 ② 새/생소한 포지션이라 **키워드 전략부터** 새로 필요할 때 ③ ClickUp 에 검색 준비가 안 된 포지션.
- 무겁다(단계 10개·게이트 다수). 급하지 않은 전방위 서치용.

### 흐름 A+ — "JD 조리"는 어디서 하나 (자주 헷갈리는 지점)

**제안 본문(JD 셋)을 발송 전에 미리 만들어 저장하는 건 /url 이 아니다.** /url 은 검색만 걸어둔다. JD 조리는 별도 스킬 3개가 채널별로 담당한다:

| 채널 | 스킬 | 부르는 말 | 산출물 |
|---|---|---|---|
| LinkedIn InMail 템플릿 | `linkedin-rps-jd-set-builder` | **"jd builder"** | RPS 템플릿 Save as new(1,899자 이내) |
| 사람인·잡코리아 이직제안 포지션 | `position-register` | "포지션 등록" / "잡코리아만 등록" | 양사 인재풀에 제안용 포지션 저장 |
| 3사 동시 | `position` | **"3사 등록: 회사, 포지션"** | 사람인+잡코리아+RPS 한 번에 |

전부 **저장·등록까지만** 하고 발송(Send)은 절대 자동으로 안 누른다(SOT3). 그래서 이상적 준비 상태 = 포지션마다 **① /url 검색 세팅 + ② JD 셋 조리** 둘 다 완료 → 후보 필요 시 /humansearch → 사장님은 보내기만.

### 흐름 C — 혼합 (실전 최다 패턴)

- **링크드인 = /url → /humansearch** 라인으로 고정, **사람인·잡코리아 = /aisearch** 가 맡는다.
- 이유: /url 은 RPS 전용이라 사람인·잡코리아엔 "미리 걸어두기"가 없다(아래 §4 갭). 그 두 채널은 /aisearch 가 검색 생성까지 처리하는 게 현재 구조상 자연스럽다.

## 3. 가장 간소하게 쓰는 법 — 외울 것 3마디

| 상황 | 한 마디 | 결과 |
|---|---|---|
| 주 1회 or 포지션 새로 들어옴 | **`/url`** | 살아있는 포지션 전부 링크드인 검색 사전 세팅 → ClickUp 저장 |
| 이 포지션 후보 보고 싶다 | **`/humansearch <회사, 역할>`** | 걸어둔 검색 순회 → 70점+ 후보 ClickUp 등록 + DM 보고 |
| 3사 전부 훑어야 한다 | **`/aisearch <포지션> 잡코리아도 같이`** | 사람인·잡코리아·RPS 병렬 풀 서치 |

- 발송은 항상 사장님이 마지막에 누른다(전 스킬 공통 불변식, SOT3).
- 개인화 제안문이 필요하면 /humansearch 뒤에 "1등 후보 제안문 만들어줘" 한 마디 추가.

## 4. 역할 경계에서 주의할 점 · 현재 갭 (반례 점검)

1. **RPS 검색 중복 위험**: /aisearch 도 RPS 검색을 만들 수 있어 /url 이 걸어둔 라이브 검색과 겹칠 수 있다(RPS 는 프로젝트당 라이브 검색 1개 — 덮어씀). → **링크드인은 /url 라인으로 통일**하고, /aisearch 는 사람인·잡코리아 위주로 부르는 걸 권장(흐름 C).
2. **/url 은 링크드인 전용**: 사람인·잡코리아용 "사전 세팅" 스킬은 없다. /humansearch 는 사람인·잡코리아 검색 URL 도 순회할 수 있지만, 그 URL 은 사람이 직접 걸거나 /aisearch 가 만들어야 한다. (개선 아이디어: /url 을 3사 사전 세팅으로 확장 — 아직 미구현, 의견일 뿐.)
3. **tier 전환은 수동**: RPS 구조상 tier(정밀·표준·확장)별 URL 저장이 안 된다(2026-07-03 검증). 댓글의 Boolean 레시피를 Keywords 에 붙여넣어 전환한다.
4. **humansearch 러너는 하드제외를 스스로 안 건다**: 등록 전 3중 게이트(영문 학교명 보정 → 하드제외 재적용 → URL 원본 복붙)가 스킬에 명문화돼 있으니 생략 금지.
5. **/aisearch 는 스스로 시작하지 않는다**: 포지션 ID·ClickUp URL·JD 중 하나는 반드시 줘야 한다.

## 4.5 포지션 인입 자동화 — 지금 있는 것 / 없는 것 (2026-07-05 코드 확인)

사장님 요구: **"Gmail 상시 감시 → 고객 메일 오면 긁어서 클릭업 등록 → 링크드인 검색 사전세팅까지 자동."**

| 단계 | 현재 상태 | 근거 |
|---|---|---|
| Gmail 상시 감시 | ❌ **없음** — 주 1회 수동 수집만(weekly-update 스킬 ①영역) | `tools/`·`scripts/`에 gmail 코드 0건(grep) |
| 포지션 → ClickUp 등록 | ✅ 있음 — 디스코드 `register-position` 명령 디스패처(FY26ClientsPosition 기본, dry-run 기본, 발송 0) | `tools/multi_position_sourcing/register_position_dispatch.py` (PR#68) |
| 등록 후 /url 자동 후속 | ❌ 없음 — 등록과 검색 세팅이 아직 이어져 있지 않음 | 디스패처는 등록에서 끝남 |

**클릭업 주소는 프롬프트에 안 붙여도 된다(이미 기본값):** /url 의 포지션 원천 = 리스트 **901814621569**(FY26ClientsPosition), /humansearch 의 후보 등록처 = 리스트 **901818680208**(FY26AI_Search) — 둘 다 스킬·설정에 박혀 있어 이름만 부르면 알아서 그 리스트를 쓴다.

## 5. 의견 (제안 — 아직 실행 안 함)

- **인입 파이프라인 완성(개발 필요)**: ① Gmail 폴링 워커(고객 메일 감지→JD 추출) → ② 기존 등록 디스패처 재사용(ClickUp 등록) → ③ 등록 성공 시 /url 사전세팅 + JD 셋 조리를 자동 후속. ①과 ③ 연결만 새로 만들면 되고 ②는 이미 있다. 원하시면 별도 작업(worktree·게이트)으로 진행.

- **기본 루틴 고정**: "포지션 들어오면 /url 자동, 후보 필요하면 /humansearch" 를 기본으로 삼고, /aisearch 는 "3사 전방위·새 키워드 전략" 전용 예약어로 좁혀 쓰면 세 스킬이 겹치지 않고 각자 최대 효용을 낸다.
- **한 마디 라우팅**: 장기적으로 "서치 <포지션명>" 한 마디가 ① ClickUp 댓글에 /url 산출물이 있으면 /humansearch, ② 없으면 /url 먼저 → /humansearch, ③ "3사/잡코리아/사람인" 단어가 있으면 /aisearch 로 자동 라우팅되면 가장 간소하다. (스킬 description 트리거 조정으로 가능 — 원하시면 별도 작업으로 진행.)
