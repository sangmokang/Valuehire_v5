3. 작업 순서

  1. PC-F4a → F4b — 자동재개 데몬. 이것 없이 "상시"가 성립 안 함(R4).
  2. PC-D1 — SQLite 저수지. 레쥬메 재활용의 척추.
  3. PC-C4a ∥ PC-N4 — 리스팅 전건 저장 / 러너 포지션 주입화 (병렬 가능).
  4. PC-D3 — pgvector 임베딩 영속 + match 라이브.
  5. PC-N3 — 상시 교차 재매칭 크론.
  6. PC-N1a → N1b — Gmail 포지션 자동 인입.
  7. PC-B3 ∥ PC-A4 — 검색 URL 저장 / 임의 URL 파서.
  8. PC-C5/C6 ∥ PC-F5 — 연봉 자산 / INV5 정리.

5. 운영 원칙 (전 조각 공통)

  • 한 조각 = 한 워크트리 = 인수기준 1개. 두 조각을 한 PR에 섞지 않는다.
  • 부품 재사용 우선. find_duplicate_position · save_rail · harvest_policy · dispatch_register_position 이 이미 있다.
  • fail-closed. 파싱 실패·장애·빈 본문을 조용히 성공 처리하지 않는다.
  • 발송은 SOT28. 어떤 조각도 "보내기"를 새로 자동화하지 않는다 — 등록·연결·드래프트까지만.
  • 착수 전 원장 회수. red-ledger 와 git log 에서 해당 PC-ID 완료 여부를 먼저 확인.
