## 0. 한 줄 요약

**디스코드에 친 한국어 문장을, 링크 없이도, 쉘 명령처럼 해석해 실행한다.** 단 해석에 실패하면 **아무것도 실행하지 않는다**.

---

## 1. 이 계약이 푸는 문제

사장님이 `클릭업에서 번개장터 PM 찾아` 라고 치시면 지금은 **입구에서 버려진다**. 이유는 분류기가 없어서가 아니라, 입구가 **URL 있는 명령만** 받도록 설계돼 있기 때문이다.

- `fleet_args.py:66-84` — 맨 토큰이 URL/스킬/머신 중 뭔지 확실치 않으면 **추측하지 않고** `None`
- `hermes_fleet_bridge.py:138` — `if not urls: return None` (본문에 URL 0개면 즉시 포기)

이 fail-closed 는 **안전장치이지 버그가 아니다.** 그래서 이 계약은 파서를 풀지 않고, **그 앞에 "이름 → URL 해소층"을 새로 단다.**

---

## 2. 문법

```
<장소>에서  <대상>  <동사>
```

3요소가 **모두** 잡혀야 한다. 하나라도 못 잡으면 **거부하고 문법을 안내**한다 — 추측해서 실행하지 않는다.

예외: **장소만** 생략되면 가장 최근 맥락의 장소를 기본값으로 쓰고, 그 선택을 한 줄로 밝힌다(CLAUDE.md §0.16 — 되묻기보다 합리적 기본값).

### 2.1 장소 어휘

| 장소 | 별칭 |
|---|---|
| `clickup` | 클릭업, clickup, 칸반, 보드 |
| `web` | 웹, web, 인터넷, 공식, 공식채용페이지, 채용페이지 |
| `saramin` | 사람인, saramin |
| `jobkorea` | 잡코리아, 잡코, jobkorea |
| `linkedin` | 링크드인, linkedin, rps, 리크루터 |
| `queue` | 큐, 작업목록, 작업, 잡, jobs |

### 2.2 동사 어휘와 위험등급

| 동사 | 별칭 | 위험등급 | 확인 필요 |
|---|---|---|---|
| `find` | 찾아, 찾아줘, 조회, 보여줘, 알려줘, 확인 | `read` | 아니오 |
| `search` | 서치, 서치해, 검색해, 후보찾아, 소싱 | `execute` | 아니오 |
| `register` | 등록, 등록해, 올려, 반영 | `write` | **예** |

**발송·삭제·계산서 발행은 어휘에 없다.** 자연어로는 발동할 수 없다(F-NL5). 슬래시 명령 전용이다.

## 3. 라우팅표 (장소 × 동사 → 무엇을 하는가)

| 장소 | 동사 | 해소기 | 실행 | 큐 스킬 |
|---|---|---|---|---|
| clickup | find | ClickUp 포지션 검색 | 작업 목록에 반영 | — |
| clickup | search | ClickUp 포지션 검색 | 큐 적재 | `aisearch` |
| **web** | **find** | **공식 채용페이지 웹 검색** | **큐 적재** | **`jdintake`** ※신규 |
| saramin | search | 검색결과 URL | 큐 적재 | `humansearch` |
| jobkorea | search | 검색결과 URL | 큐 적재 | `humansearch` |
| linkedin | search | RPS 라이브 검색 | 큐 적재 | `url` |
| queue | find | — | 작업 상태 회신 | — |

> ※ `jdintake` 는 **아직 없는 스킬**이다. 현재 큐가 받는 스킬은 `humansearch/aisearch/url/agent` 4종뿐(`job_queue.py:26-29`). 요구 N3("웹에서 JD 파악")은 명령 배선이 아니라 **스킬 신규 개발**이며 AC-N3 로 분리한다.

## 4. 해소 정책 — 몇 건 나왔느냐로 갈린다

| 결과 | 동작 | 실행해도 되는가 |
|---|---|---|
| **0건** | "못 찾았습니다" 회신 후 종료 | **금지.** 다른 장소로 임의 확장도 금지 |
| **1건** | "이렇게 이해했습니다" 1줄 표기 후 진행 | 허용 |
| **N건** | 선택지 최대 4개 제시 | **금지** — 사장님이 고르신 뒤 진행 |

**N건 질문은 되묻기 금지(§0.16) 위반이 아니다.** "할까요?"(실행 동의)는 금지지만 **"어느 것?"(갈림길)** 은 명시적으로 허용된 형태다. 실제로 `번개장터 PM` 은 ClickUp 에 Core Product / Business Development 2건이 존재한다.

## 7. Hook 강제 (SOT-27 규약)

`.claude/hooks/guards/nl-shell-routing.py` 한 파일. `harness-dispatch.py` 가 자동 발견해 **모든 툴 호출**에 문을 건다. 디스패처는 수정하지 않는다.

| H | 막는 것 |
|---|---|
| H-NL1 | 정본 모듈 밖 즉석 자연어 파서·라우팅 스크립트 자작 (F-NL4) |
| H-NL2 | 대상 미해소 상태의 실행형 큐 적재 (F-NL3) |
| H-NL3 | 자연어 경로에서 포털 raw 조작·발송으로 새는 것 (F-NL1·F-NL2) |
| H-NL4 | 이 문서/JSON 계약과 코드 어휘의 드리프트 |

가드는 **fail-open** 이다(로드 실패 시 통과). 따라서 보안을 가드 하나에 걸지 않고, 해소층 코드 안의 정책(§4)과 **이중**으로 건다.

### 7.1 활성 조건 — 아직 잠들어 있다 (2026-07-22 V1 적대검증 결과)

가드는 환경변수 **`VH_NL_SHELL`** 이 켜진 컨텍스트에서만 문을 건다. **현재 이 변수를 켜는 코드는 저장소에 없다** — 즉 가드는 문법·테스트상 완성됐지만 **운영에서는 아직 발동하지 않는다.**

이는 순서상 정상이다: 자연어 처리 경로(`nl_shell.py`)가 AC-N1 에서 만들어지고, 그때 하위 프로세스 env 에 `VH_NL_SHELL=1` 을 주입해야 비로소 살아난다(`fleet_worker.py:819` 의 `VH_BUSY_TASK` 주입과 동일 패턴). **AC-N1 의 완료 판정에 이 배선 단언이 포함된다.**

> 정직 표기: 이 가드를 두고 "지금 보호되고 있다"고 말하면 거짓이다. 계약과 문이 준비된 상태이며, 문을 여는 스위치는 AC-N1 에서 연결된다.
