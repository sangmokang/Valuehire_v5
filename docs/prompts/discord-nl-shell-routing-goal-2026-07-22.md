## 0. 사장님께 한 줄 (쉬운 말)

사장님이 디스코드에 **"클릭업에서 번개장터 PM 찾아"** 라고 치시면, 지금은 아무 일도 안 일어납니다. 링크(URL)를 안 붙이셨다는 이유로 입구에서 버려지기 때문입니다. 이 작업은 **사장님 말에서 "어디서 / 무엇을 / 어떻게"를 뽑아내, 링크를 봇이 스스로 찾아 붙이고** 그 다음 기존 검색 경로에 태우는 것입니다. 즉 디스코드가 **터미널(쉘창)처럼** 동작하게 만듭니다.

## 2. 지금 왜 안 되는가 — 실측 진단 (추측 아님, 2026-07-22)

사장님이 디스코드에 `클릭업에서 번개장터 pm` 을 보내셨을 때 실제로 일어난 일을 코드로 역추적한 결과다.

| # | 사실 | 증거 |
|---|---|---|
| B1 | 메시지는 먼저 `key:value` 파서를 통과해야 한다. 맨 토큰이 URL/스킬/머신 중 하나로 확실히 안 잡히면 **추측하지 않고 거부**한다 | `tools/multi_position_sourcing/fleet_args.py:66-84`(`_classify_bare_fleet_run_token` → 모호하면 `None`), `:144`(`형식 오류(키:값 아님)`) |
| B2 | 자연어 우회로(`_natural_fleet_command`)도 **본문에 URL이 하나도 없으면 즉시 포기**한다 | `tools/multi_position_sourcing/hermes_fleet_bridge.py:138` (`if not urls: return None`) |
| B3 | 그래서 `/fleet-run` 잡이 **생성되지 않았고**, 로그인·브라우저 검색 경로가 아예 발사되지 않았다 | 디스코드 로그: `거부됨: 형식 오류(키:값 아님): '클릭업에서'` → 이후 잡 번호 없음 |
| B4 | 대조군: 같은 날 ClickUp URL 이 붙은 메시지는 **정상 접수**됐다 | 디스코드 로그 `job #59 machine=macmini skill=url followup_skill=aisearch` |
| B5 | 큐가 받아주는 스킬은 **4종뿐**이다 | `job_queue.py:26-29` → `FLEET_SKILLS=("humansearch","aisearch","url")`, `OWNER_AGENT_SKILL="agent"` |
| B6 | **"웹에서 JD 파악"에 해당하는 스킬이 없다** | 위 4종 어디에도 없음. `jdbuilder` 는 등록 스킬이지 탐색 스킬이 아님(SOT-25) |
| B7 | 봇은 **Message Content 인텐트가 꺼져 있다** — 서버 채널의 평문은 봇에게 배달되지 않는다. DM·멘션만 내용이 보인다 | `scripts/discord_direct_gateway.py:541,553` (`discord.Intents.default()`), `discord_routing.py:194-204` (의도적 설계) |

### 2.1 진단 요약

- **근본 원인은 "분류기가 없다"가 아니다.** 입구가 **URL을 가진 명령만** 통과시키도록 설계돼 있고(B1·B2), 그건 안전을 위한 의도적 fail-closed 다.
- 따라서 고칠 지점은 파서를 **느슨하게 푸는 것이 아니라**, 그 **앞단에 "이름 → URL 해소(resolve)" 층을 새로 다는 것**이다. 파서의 fail-closed 는 그대로 둔다.
- 추가로 **N3(웹 JD 파악)은 능력 자체가 없다**(B6). 명령을 붙이는 일이 아니라 **스킬을 새로 만드는 일**이다.
- **B7은 사장님 사용 방식에 영향을 준다** — DM 이면 지금도 평문이 보인다. 서버 채널 평문까지 원하시면 Discord 개발자 포털에서 특권 인텐트를 켜야 한다(§9 E-NL4).

## 3. 설계 원칙 (기존 것을 부수지 않는다)

1. **파서를 풀지 않는다.** `fleet_args.parse_fleet_args` 의 fail-closed(모호하면 거부)는 **그대로 유지**한다. 자연어는 그 앞에서 해소되어, 파서에는 항상 **완성된 `key:value` 명령**만 들어간다.
2. **해소(resolve)와 실행(execute)을 분리한다.** 해소가 실패하면 실행은 **절대** 일어나지 않는다. 추측 실행 금지(CLAUDE.md §0.2).
3. **새 러너를 만들지 않는다.** 해소 결과는 기존 `/fleet-run <url> …` 계약으로 되돌려, 기존 큐·워커·스킬을 그대로 탄다(SOT-25 단일 진입점 원칙 연장).
4. **되묻기는 최소, 그러나 대상이 여럿이면 고른다.** CLAUDE.md §0.16 "실행 동의 되묻기 금지"는 유지 — 다만 **"할까요?"가 아니라 "어느 것?"** 은 허용된 갈림길 질문이다.
5. **Hook 으로 이중 강제.** 문서 규칙은 위장 가능하므로, 결정적 차단은 harness 가드가 건다(SOT-27 규약, `.claude/hooks/harness-dispatch.py`).

## 4. 목표 문법 — "쉘창처럼"

```
<장소>에서  <대상>  <동사>
   │         │       └─ 찾아 / 서치 / 등록 / 보여줘 …
   │         └───────── 회사명 + 직무 (자유 텍스트)
   └─────────────────── 클릭업 / 웹 / 사람인 / 잡코리아 / 링크드인 / 큐
```

사장님 예시가 이 문법에 그대로 앉는다:

| 사장님 입력 | 장소 | 대상 | 동사 | 해소 결과 | 최종 실행 |
|---|---|---|---|---|---|
| `클릭업에서 번개장터 PM 찾아` | `clickup` | 번개장터 / PM | `find` | ClickUp Task URL | 작업 목록에 반영 (N2) |
| `웹에서 공식 채용페이지에서 번개장터 PM 찾아` | `web` | 번개장터 / PM | `find` | 공식 채용페이지 URL | JD 파악·요약 (N3) |
| `클릭업에서 번개장터 PM 서치해` | `clickup` | 번개장터 / PM | `search` | ClickUp Task URL | `fleet-run aisearch <url>` (기존 경로) |

**장소가 생략되면** 가장 최근 맥락의 장소를 기본값으로 쓰고, 그 선택을 한 줄로 밝힌다(CLAUDE.md §0.16).

## 5. 목표 아키텍처 — 앞단에 해소층을 단다

```
디스코드 평문 (DM · 멘션)
        │
        ▼
┌───────────────────────────────────────────────┐
│ ① NL 문법 파서  nl_shell.parse()               │  ← 신규 (규칙 기반, LLM 호출 0)
│    "클릭업에서 번개장터 PM 찾아"                │
│      → (locus=clickup, target="번개장터 PM",   │
│         verb=find)                             │
└───────┬───────────────────────────────────────┘
        │ 3요소가 안 잡히면 → 거부 + 문법 안내 (추측 금지)
        ▼
┌───────────────────────────────────────────────┐
│ ② 대상 해소기  nl_shell.resolve()              │  ← 신규. locus 별 해소 어댑터
│    clickup → ClickUp 검색 API → Task URL       │
│    web     → 웹 검색 → 공식 채용페이지 URL      │
│    saramin/jobkorea/linkedin → 기존 검색 URL    │
└───────┬───────────────────────────────────────┘
        │ 0건 → "못 찾았습니다" (실행 금지)
        │ N건 → 선택지 버튼 (갈림길 질문, 허용)
        │ 1건 → 확정
        ▼
┌───────────────────────────────────────────────┐
│ ③ 기존 계약으로 환원                            │  ← 재사용, 신규 코드 최소
│    "/fleet-run <skill> <url> channels:…"       │
│    → fleet_args.parse_fleet_args() (무수정)     │
└───────┬───────────────────────────────────────┘
        ▼
   기존 큐 public.jobs → fleet_worker → 결과 회신
```

**핵심**: ③ 이후는 **한 줄도 바뀌지 않는다**. 이번 작업의 신규 코드는 ①②와 가드뿐이다.

## 7. Spec(SOT) 로 무엇을 못박는가

정본: **`docs/sot/32-nl-shell-routing.md`** (사람이 읽는 계약) + **`docs/sot/32-nl-shell-routing.json`** (기계가 읽는 계약).

JSON 계약이 단일 출처로 고정하는 것:

| 키 | 내용 | 왜 기계 계약인가 |
|---|---|---|
| `loci` | 장소 어휘 + 별칭(클릭업/clickup, 웹/web …) | 문서와 코드가 따로 놀지 않게 (드리프트 0) |
| `verbs` | 동사 어휘 + 각 동사의 위험등급(read/execute/write) | 위험등급이 확인 게이트 필요 여부를 결정 |
| `routes` | (장소 × 동사) → 대상 스킬·해소 어댑터 | 라우팅표를 코드에 흩뿌리지 않음 |
| `resolution_policy` | 0건·1건·N건일 때의 동작 | "추측 실행 금지"를 기계가 검사 가능하게 |
| `forbidden` | 자연어가 절대 도달하면 안 되는 행동 | 발송·삭제·즉석 포털 조작 |

## 8. Hook 설계 (N5 — "Spec에 명확히 Hook으로 말아줘")

`.claude/hooks/guards/nl-shell-routing.py` 1개 파일. `harness-dispatch.py` 가 자동 발견해 **모든 툴 호출**에 적용한다(SOT-27 규약, 디스패처 수정 불필요). 가드는 fail-open 이므로 §7 계약과 **이중**으로 건다.

| H | 막는 것 | 왜 |
|---|---|---|
| **H-NL1** | 정본 모듈(`tools/multi_position_sourcing/nl_shell.py`) 밖에서 **즉석 자연어 파서·라우팅 스크립트 자작** | 2026-07-09 지아이텍 사고 유형(즉석 경로 우선)의 재발 방지. 새 러너 금지 원칙(CLAUDE.md §0.2) |
| **H-NL2** | **대상 미해소 상태의 실행형 큐 적재** — `resolve` 산출 URL 없이 `fleet-run`/enqueue 를 부르는 호출 | 원칙 2(추측 실행 금지)를 기계로 강제 |
| **H-NL3** | 자연어 경로에서 **채용사이트 raw 조작·제안 발송**으로 새는 것 | SOT-25 §0 연장. 자연어가 우회로가 되면 안 됨 |
| **H-NL4** | SOT 32 JSON 계약과 코드 어휘(`loci`/`verbs`)의 **드리프트** | 문서만 고치고 코드는 안 고치는 위장 방지 |

각 가드는 `.claude/hooks/tests/` 에 대응 테스트를 둔다(기존 `test_runner_lease_guard.py` 패턴).
