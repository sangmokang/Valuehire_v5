## 1. 이슈 S1 — 채점 재보정 (한국형 신호 + 죽은 배점 부활)

### 확정 설계
1. **학위(R1)**: 한글 토큰("학사","석사","박사","공학","이학","전문학사" 제외) 추가 — 학사+ 는 10점.
   전문대/전문학사는 기존 하드제외 규칙 유지(완화 아님).
2. **인서울 대학 티어(R2)**: `MID_TIER_SCHOOL_SIGNALS`(신규) — 인서울 4년제
   (중앙·경희·한국외대·서울시립대·건국·동국·홍익·숙명·이화·국민·세종·숭실·단국(죽전 포함 표기 주의)·
   광운·명지·상명·성신·덕성·동덕·서울여·한성·삼육·가톨릭) + 지방 거점 국립(부산·경북·전남·전북·충남·충북)
   → **5점**. 기존 HIGH_TIER(8점)와 배타 적용(중복 가점 없음). 영문 표기(예: "Chung-Ang University")도 신호에 포함.
3. **산업/스테이지(R3)**: 15점 → 5점으로 축소하고, 남는 10점을 필수요건(40→45)·연차(15→20)로 재배분.
   (문자열 그대로 매칭이라는 사문 조건은 유지하되 비중만 축소 — 판정 로직 신규 발명 금지, 최소 변경.)
4. **연차(R4)**: years_experience 미파싱 시 0 대신, employment_history 재직기간 합산으로 추정
   (기존 tenure_months 재사용). 추정치 사용 시 why_fit 에 "연차 추정" 명시.
5. **커트라인 70 유지.** 재보정 후 도달 가능해지므로 커트라인은 건드리지 않는다(합격 기준 약화 금지).
6. **60~69점 밴드**: 미등록 폐기 대신 ClickUp 설명/Discord 리포트에 "검토 후보(보류)" 섹션으로
   이름+점수+1줄만 별도 표기(Subtask 생성은 여전히 70+만 — 등록 게이트 불변).

### 인수 기준 (pytest — tests/test_scoring*.py)
- 학력란 "OO대학교 학사"(한글만) → education 10점
- 교육란 "중앙대학교"(또는 "Chung-Ang University") → university_tier 5점, "서울대"는 여전히 8점(중복 없음)
- breakdown 합계 검증: stage_industry ≤ 5, must_have ≤ 45, seniority ≤ 20 (총합 100 유지)
- years_experience=None + 재직이력 합산 84개월 → 연차 추정으로 seniority > 0
- 기존 하드제외(프리랜서·단기이직 2회+·전문대) 테스트 전부 회귀 없음
- 재보정 후 "인서울+한글학위+연차파싱" 표준 프로필 픽스처가 70점 이상 도달(구조적 도달 가능성 증명)

### 게이트 2 프롬프트
```
make task NAME=humansearch-scoring-recalibration
위 인수 기준 RED→GREEN. 파일: tools/multi_position_sourcing/scoring.py + tests.
목표 규모: 파일 2개, diff 80~250줄. ./verify.sh 출력 그대로. make ship.
⚠️ 하드제외·70 커트라인·등록 게이트(eligible_matches_for_send)는 절대 약화 금지.
```

## 2. 이슈 S2 — 인서울 대학 프로필 전수 열람 (열람 트리아지 규칙)

### 확정 설계 (SKILL.md 절차 + 러너 로직)
- 리스트(수확 카드) 단계에서 **열람 우선순위 큐**를 만든다:
  - **T1(무조건 열람·스킵 금지)**: 카드 ctx(학력/헤드라인)에 인서울·거점국립·해외명문 대학 신호.
  - **T2(기본 열람)**: 대학 신호 미노출(리스트에 학력 안 보이는 경우 포함 — 안 보인다고 스킵 금지).
  - **T3(최후순위)**: 이슈 S3 의 해외 후보 신호.
- 페이지 커버리지 기존 규칙(최소 10페이지 또는 마지막 페이지) 유지 + **T1 잔존 카드가 있는 한
  열람 중단 금지**를 명문화(러너가 시간/개수 예산으로 조기 종료할 때도 T1 은 소진 후 종료).

### 인수 기준
- 트리아지 순수함수: 카드 목록 입력 → T1/T2/T3 정렬 출력(인서울 신호 카드가 항상 앞).
- "학력 미노출 카드"가 T2(열람 대상)로 분류됨 — 스킵 아님.
- 예산 축소 시나리오 mock: T1 카드가 남아 있으면 T2/T3 부터 잘리고 T1 은 전부 열람.

## 3. 이슈 S3 — 해외(비한국) 후보 열람 최후순위 (사장님 지시 반영)

### 확정 설계 — "이름 그 자체"가 아니라 **한국 근무 적합 신호**로 판정한다
지시 취지 = 국내 포지션에 맞지 않는 해외 후보에게 열람 시간을 쓰지 말 것. 판정은 이름 대신
아래 신호 조합으로 한다(이름 단독 판정은 오탐·차별 리스크 — 예: 한국 대학 나온 외국 국적 개발자,
영문 표기 쓰는 한국인):
- **T3(최후순위) 강등 조건**: 다음 신호가 **하나도 없을 때** — ①한글 표기(이름/헤드라인/자기소개)
  ②한국 소재(Seoul/Korea 등 location) ③한국 대학 학력 ④한국 회사 재직 이력.
- T3 는 "거의 안 봐도 됨" = T1·T2 소진 후 잔여 예산에서만 열람. 아예 0 으로 만들지는 않는다
  (JD 가 영어 가능/글로벌 채용을 명시하면 이 규칙 자체를 끈다 — JD 플래그로 제어).
- 채점기(scoring.py)의 korea_language_region 5점은 그대로 두고, **열람 순서만** 제어한다
  (점수 조작이 아니라 시간 배분 문제로 푼다).

### 인수 기준
- 순수함수 `korea_fit_tier(card)` : 한글 이름 카드 → T2 이상, 4신호 전무 카드 → T3,
  "외국 이름 + 한국 대학" 카드 → T2 (이름만으로 강등되지 않음을 명시 고정)
- JD 에 global/english-ok 플래그 → 전 카드 T3 강등 없음
