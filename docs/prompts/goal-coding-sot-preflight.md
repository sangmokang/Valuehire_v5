# 코딩용 Goal Prompt — SOT 먼저, 코딩은 그 다음

> 언제 쓰나: 새 기능 구현, 버그 수정, 스킬 수정, 자동화 수정, 외부 연동 수정처럼
> **코드를 쓰거나 파일을 바꾸는 모든 작업**을 시작할 때.
>
> 목적: 모델이 SOT를 무시하고 임의로 새 코드·새 러너·새 등록 스크립트를 만드는 일을 막는다.
> 이 프롬프트는 `CLAUDE.md`, `docs/harness.md`, 관련 `docs/sot/`보다 약하지 않다.

---

## 그대로 붙여 넣을 프롬프트

```text
/st

너는 지금부터 Valuehire_v5에서 코딩 작업을 한다.

가장 중요한 원칙:
작업·코딩·브라우저 조작·외부 쓰기 전에 먼저 기존 정의를 회수한다.
이 단계를 건너뛰면 SOT 위반이다.

작업 목표:
[여기에 작업 목표를 쓴다]

먼저 해야 할 일:
아직 코딩하지 마라.
아직 파일 만들지 마라.
아직 브라우저 조작하지 마라.
아직 Discord, ClickUp, 이메일, 채용사이트에 쓰지 마라.

반드시 먼저 읽고 보고:
1. 루트 SOT: CLAUDE.md
2. 작업 루프: docs/harness.md
3. 관련 SOT: docs/sot/
4. 관련 스킬: skills/*/SKILL.md 또는 ~/.codex/skills/*/SKILL.md
5. 관련 config/json
6. 기존 구현 진입점
7. 과거 메모리·로그·기존 구현 검색 결과

먼저 보고할 5가지:
1. 읽은 경로
2. 기존 구현 진입점
3. 재사용·확장할 파일/함수
4. 새 파일 필요 여부와 이유
5. 외부 쓰기 여부와 승인 게이트

강제 금지:
- 기존 정의·구현 회수 전 새 코드 작성 금지
- 기존 경로로 가능한데 새 파일 작성 금지
- 기존 경로로 가능한데 새 러너 작성 금지
- 기존 경로로 가능한데 새 등록 스크립트 작성 금지
- 스펙을 사후에 추가해 현재 행동을 정당화 금지
- 추측 진행 금지
- 테스트 약화·삭제 금지
- main 직접 수정 금지

STOP 조건:
- 기존 정의를 못 찾음
- 기존 구현 진입점을 못 찾음
- SOT와 사용자 요청이 충돌함
- 참조된 파일/스크립트가 없음
- 로그인, 캡차, 봇차단, 2FA, checkpoint 발생
- 외부 쓰기가 필요함
- 프로필 URL, 포지션 URL, 필수 필드 무결성을 확인하지 못함

STOP 조건이면 진행하지 말고 멈춰서 보고한다.
임시 우회, 추측 구현, 사후 스펙 추가로 해결한 척하지 않는다.

외부 쓰기는 항상 L3:
- Discord 메시지
- ClickUp 댓글, 필드 업데이트, 태스크 생성
- 이메일 발송
- 채용사이트 게시, 등록, 저장, 제안, InMail
- 사람인·잡코리아·LinkedIn/RPS 쓰기

외부 쓰기는 사장님 명시 승인 전까지 dry-run, 초안, 저장 계획까지만 한다.
승인 요청 시에는 어디에, 무엇을, 몇 번 쓰는지와 알람 발생 가능성을 정확히 보고한다.
알람 폭탄 금지. 여러 후보·여러 포지션·여러 항목은 한 메시지 또는 한 댓글로 묶는다.

구현 방식:
1. 현재 상태 증거를 file:line으로 잡는다.
2. RED 테스트 또는 재현 명령을 먼저 만든다.
3. 최소 변경으로 GREEN을 만든다.
4. 프로덕션 호출 경로에 실제 배선됐는지 확인한다.
5. 내 코드를 두 번 깬다.
   - 1차: 내가 직접 빈 값, 잘못된 입력, 중복, 막힌 사이트, secret 노출, URL 오류를 넣어 공격한다.
   - 2차: 가능한 경우 독립 검증자에게 같은 diff를 적대적으로 검토시킨다.
6. 검증 명령의 실제 출력만 보고한다. 지어내지 않는다.

완료 보고 형식:
- Summary
- Files Changed
- Verification
- Assumptions / Risks

사장님께는 한국어로 짧고 쉽게 말한다.
어려운 내부 용어를 쓰면 바로 옆에 쉬운 설명을 붙인다.

이제 작업하지 말고, 먼저 기존 정의 회수 결과부터 보고해라.
```

## 이 문서를 바꿀 때의 기준

- 이 문서는 코딩 작업 시작 게이트다. 약하게 고치지 않는다.
- `CLAUDE.md`의 “시작 전 과거 지시 회수”와 `docs/harness.md`의 게이트 0.5를 반복해서 강제한다.
- 새 코드보다 기존 구현 재사용이 우선이다.
- 외부 쓰기는 항상 L3이며, 명시 승인 전에는 dry-run/초안/계획까지만 허용한다.
- 이 문서의 핵심 문구는 `tests/test_goal_coding_sot_preflight_prompt.py`가 감시한다.
