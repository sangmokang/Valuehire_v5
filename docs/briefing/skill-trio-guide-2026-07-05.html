서치 스킬 3형제 가이드

/url · /aisearch · /humansearch — 역할 분담과 가장 간소한 사용법 · 2026-07-05
"/url 은 밥상 차리기, /humansearch 는 밥 먹기, /aisearch 는 장보기부터 풀코스."

1. 세 스킬의 역할 (딱 갈라서)

준비

/url

  • 링크드인(RPS) 검색을 미리 걸어둔다
  • JD에서 검색식 3단(정밀·표준·확장) 생성
  • 라이브 검색 링크 1개 + 레시피를 ClickUp 포지션 댓글에 저장
  • 인자 없이 치면 살아있는 포지션 전수 세팅
후보 열람·채점·발송은 안 함 · 링크드인 전용
수확

/humansearch

  • 걸어둔 검색결과(RPS·사람인·잡코리아)를 순회·채점
  • 합격 70점+ 후보를 ClickUp FY26AI_Search 에 등록
  • 전부 저장 + 사장님 DM 보고
  • 요청 시 1등 후보 개인화 제안문까지
검색어 생성·필터 입력은 안 함 · 발송(Send)은 절대 자동 안 누름
풀코스

/aisearch

  • JD 분석 → 키워드 5축 전략
  • 사람인·잡코리아·링크드인 3사 병렬 검색
  • 적합도 평가 → 표준 후보 리스트(링크·점수·이유·요약)
포지션 입력 없이는 시작 안 함 · 무겁다(전방위용) · 발송 자동 금지

핵심 경계: 검색을 "만드는" 건 /url(링크드인)과 /aisearch(3사), 검색을 "소비하는" 건 /humansearch.

2. 권장 흐름 — 링크드인 기본 루틴 (가장 간소)

1
/url — 포지션이 새로 들어오거나 주 1회. 살아있는 포지션 전부 링크드인 검색을 미리 세팅해 ClickUp 에 저장. (밤에 돌려도 됨)
2
/humansearch 회사, 역할 — 후보가 필요한 순간 포지션 이름만 부른다. 순회·채점·등록·DM 보고까지 한 턴.
3
사장님이 직접 발송 — 제안·InMail 의 "보내기"는 항상 사람 손 (모든 스킬 공통 철칙).

준비(비싼 작업)와 수확(빠른 작업)이 분리돼 있어서, 필요한 순간엔 한 마디면 됩니다. 사람인·잡코리아까지 봐야 하면 그때 /aisearch 를 부릅니다.

2.5 "JD 조리"는 어디서? (자주 헷갈리는 지점)

제안 본문을 발송 전에 미리 만들어 저장하는 건 /url 이 아닙니다. /url 은 검색만 걸어둡니다. JD 조리는 채널별 전담 스킬이 따로 있어요:

채널부르는 말결과
링크드인 InMail 템플릿jd builderRPS 템플릿 저장(Save as new, 1,899자 이내)
사람인·잡코리아 이직제안포지션 등록 (잡코리아만/사람인만 지정 가능)양사 인재풀에 제안용 포지션 저장
3사 동시3사 등록: 회사, 포지션사람인+잡코리아+링크드인 한 번에

전부 저장까지만 — 보내기는 항상 사장님 손. 그래서 이상적 준비 상태 = 포지션마다 ① /url 검색 세팅 + ② JD 셋 조리 둘 다 완료 → 후보 필요 시 /humansearch → 사장님은 보내기만.

2.7 포지션 인입 자동화 — 지금 있는 것 / 없는 것

목표 그림: Gmail 상시 감시 → 고객 메일 오면 긁어서 클릭업 등록 → 링크드인 검색 사전세팅까지 자동. (2026-07-05 코드 확인 결과)

단계상태비고
Gmail 상시 감시❌ 없음주 1회 수동 수집만(위클리 준비 때). 상시 감시 워커는 미개발
포지션 → 클릭업 등록✅ 있음디스코드 명령 한 줄로 등록되는 장치가 지난주 완성(발송 0, 안전 기본)
등록 후 /url 자동 후속❌ 없음등록과 검색 세팅이 아직 이어져 있지 않음
클릭업 주소는 안 붙여도 됩니다(이미 기본값) — 포지션 리스트(901814621569)와 후보 결과 리스트(901818680208)는 스킬 설정에 박혀 있어서, 프롬프트에 URL 없이 이름만 부르면 알아서 그 리스트를 씁니다.

4. 주의할 점 (지금 구조의 한계)

링크드인 검색 겹침 주의 — /aisearch 도 링크드인 검색을 만들 수 있는데, RPS 는 프로젝트당 라이브 검색이 1개뿐이라 /url 이 걸어둔 검색을 덮을 수 있습니다. → 링크드인은 /url→/humansearch 라인으로 통일, /aisearch 는 사람인·잡코리아 위주로 쓰는 걸 권장.
/url 은 링크드인 전용 — 사람인·잡코리아용 "미리 걸어두기"는 아직 없습니다. 그 두 채널의 검색 생성은 /aisearch 몫.
검색 단계(tier) 전환은 수동 — 정밀→표준→확장 전환은 ClickUp 댓글의 검색식을 복사해 붙여넣는 방식(링크드인 구조상 URL 여러 개 저장 불가).
등록 전 3중 확인 생략 금지 — 영문 학교명 보정 → 하드제외(프리랜서·잦은 이직) 재적용 → 프로필 링크 원본 복붙. /humansearch 에 명문화돼 있음.

5. 의견 (제안)

인입 파이프라인 완성(개발 필요) — ① Gmail 감시 워커(고객 메일 감지→JD 추출) → ② 이미 있는 클릭업 등록 장치 재사용 → ③ 등록되면 /url 검색 세팅 + JD 셋 조리 자동 후속. 새로 만들 건 ①과 ③의 연결뿐입니다. 원하시면 별도 작업으로 진행합니다.
역할 고정 — "포지션 오면 /url, 후보 필요하면 /humansearch"를 기본 루틴으로, /aisearch 는 "3사 전방위·새 키워드 전략" 전용으로 좁혀 쓰면 세 스킬이 겹치지 않고 각자 최대 효용을 냅니다.
한 마디 라우팅(향후) — "서치 코드잇 백엔드" 한 마디로: 준비돼 있으면 /humansearch, 없으면 /url 먼저 자동, "3사" 단어 있으면 /aisearch. 스킬 트리거 조정으로 가능 — 원하시면 별도 작업으로 진행합니다.
