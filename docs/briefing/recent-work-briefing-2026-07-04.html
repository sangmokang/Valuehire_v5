7발송 자동화 로드맵 — 반자동을 거쳐 자동으로 (플랜)

지금은 "보내기 = 항상 사장님 손"이 회사 규칙입니다. 이 규칙은 사장님이 명시적으로 바꾸기 전까지 그대로이고, 아래는 그날을 위해 미리 그려두는 단계별 전환 계획입니다. 핵심 원칙: 자동화 수준은 무사고 실적으로만 한 단계씩 올라간다.

단계 0 — 지금 (수동)문구 생성·검문은 자동, 붙여넣기·Send는 사람. 오늘 기준 여기까지 완성.
→
단계 1 — 반자동 (다음 목표)시스템이 "보낼 준비 완료" 대기열을 만들고, 사장님은 승인 한 번. 입력·대조까지 자동, Send만 사람.
→
단계 2 — 조건부 자동 (추후)규칙 개정 후, 승인된 범위 안에서만 자동 발송 + 전건 사후보고.

🟡 단계 1 — 반자동 "승인 대기열" 설계

사장님의 클릭을 3번(읽기→붙여넣기→Send)에서 2번(승인→Send)으로 줄이는 단계.

  • 흐름: humansearch가 핵심 후보 선정 → 생성기(PC-G2)가 문구 조립 → 검문소 통과분만 Discord DM으로 "발송 대기" 카드 도착(후보·문구·점수·근거) → 사장님이 "승인"이라고 답하면 → 시스템이 컴포저에 자동 입력 + 화면 스크린샷 대조까지 마치고 → Send 클릭만 사장님
  • 재사용 부품: 문구 생성(PC-G2)·검문소(precheck)·글자수 캡(PC-G1)·Discord 보고 경로(dm_report)·페이싱(PC-E1) — 새로 만드는 건 대기열·승인 명령·컴포저 자동입력 3개
  • 선행 조건(코드): ① 캡 초과 STOP 테스트 강화(PC-G2b) ② 생성기를 humansearch 절차에 배선 ③ 이름검사 오탐 수리(#60) ④ 컴포저 입력 후 스크린샷-원문 자동 대조
  • 안전: 승인 없이 입력 자체를 안 함 · 입력 후 대조 불일치면 자동 폐기 + 보고 · 승인 유효시간(예: 24시간) 지나면 자동 만료

🔵 단계 2 — 조건부 자동 발송 (규칙 개정 필요)

아래 승격 조건을 전부 충족하고, 사장님이 회사 규칙(보내기=사람)을 명시적으로 개정한 뒤에만 시작.

  • 승격 조건(기계로 셀 수 있는 것): 반자동 운영 30일 + 오발송 0건 + 대조 불일치 자동차단 실적 확인 + #60 수리 완료
  • 범위 제한: 사장님이 포지션 단위로 사전 승인한 것만 자동 발송(화이트리스트). 신규 포지션은 항상 반자동부터
  • 가드레일: 일일 발송 상한(채널별) · 사람 같은 간격(PC-E1 재사용) · 발송 직전 검문 재실행 · 킬스위치 한 마디("자동발송 꺼")로 즉시 전체 중단 · 전건 Discord 사후보고(누구에게·언제·몇 자)
  • 실패 시: 어떤 오류든 그 채널 자동발송 전체 정지 + 원인 보고 후 사람 재승인 전 재개 금지 (fail-closed)

⚠️ 자동 발송의 리스크 — 미리 알고 가야 하는 것

리스크내용대비
오발송다른 후보 이름·회사가 섞인 메일이 나감 (Movensys 사고 유형)이중 검문(생성 시 + 발송 직전) + 스크린샷 대조 + #60 이름검사 수리
계정 제재LinkedIn 등은 자동 발송 도구에 계정 제한을 걸 수 있음(약관)사람 간격 페이싱 + 일일 상한 + 차단신호 즉시 STOP. 단계 2 진입 전 채널별 약관 리스크 별도 검토 필수
평판기계 냄새 나는 대량 메일 → 회신율 하락·스팸 신고개인화 오프너 필수(생성기 구조가 강제) + 하루 소량 + 핵심 후보만
회수 불가발송은 되돌릴 수 없는 동작그래서 승격은 실적으로만, 범위는 화이트리스트로만, 중단은 한 마디로

요약: 반자동(단계 1)은 지금 부품으로 곧 갈 수 있고 안전 이득이 큽니다. 자동(단계 2)은 실적 + 규칙 개정 + 약관 검토라는 3중 문을 다 열어야 갑니다.
