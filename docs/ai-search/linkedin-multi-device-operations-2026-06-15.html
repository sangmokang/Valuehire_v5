계정 3개 · 맥 2대, 누가 어디서 돌릴까

맥북에어 + 맥미니 두 대로 링크드인 계정 3개(본계정 + 부계정 2)를 안 막히게 나눠서 운영하는 방법입니다. 핵심은 "기기를 나누는 것"보다 "계정마다 브라우저를 완전히 분리하고, 동시에 켜지 않는 것"입니다.

작성일 2026-06-15 기기 MacBook Air + Mac mini 계정 3개 발송 사람이(SOT 3번)

한 줄 요약

"한 기기에 한 계정"은 계정 3개를 2대에 못 담아서 안 맞습니다. 대신 본계정은 맥북에어에 독립, 부계정 2개는 맥미니에서 시간을 나눠 돌립니다.

왜 이렇게? → 두 맥이 같은 와이파이면 링크드인엔 IP가 어차피 같습니다. 그래서 진짜 중요한 건 ① 계정마다 크롬 프로필을 따로 쓰고 ② 두 계정을 동시에 켜지 않고 ③ 가장 소중한 본계정은 한 대에만 두는 것입니다.

00 — 원리

링크드인은 무엇을 보고 "같은 사람"이라 묶나

부계정 운영이 위험한 건, 링크드인이 여러 신호로 "이 계정들 한 사람이네"라고 묶어서 같이 제재하기 때문입니다. 묶는 신호는 아래 4가지인데, 우리가 손댈 수 있는 건 정해져 있어요.

🌐 IP 주소 — 못 바꿈

같은 와이파이면 두 맥 다 같은 IP. 하지만 사무실 IP를 여러 사람이 쓰는 건 정상이라 이것만으론 잘 안 막힙니다.

🍪 브라우저·쿠키 — 우리가 분리 가능 ★

계정마다 크롬 프로필을 따로 쓰면 쿠키·지문이 갈라집니다. 제일 효과 큰 방어.

⏰ 동시 접속 — 우리가 조절 가능 ★

두 계정이 같은 시간에 같은 기기/IP에서 활동하면 묶입니다. 시간을 나누면 됩니다.

👤 행동 패턴 — 우리가 다르게 ★

세 계정이 똑같은 시간·똑같은 메모·똑같은 대상에 신청하면 봇으로 보입니다. 다르게.

결론 — IP는 못 바꾸지만(그리고 사무실 IP 공유는 정상이라 큰 문제 아님), 브라우저 분리 + 시간 분리 + 행동 다르게 이 3개만 지키면 같은 IP라도 충분히 안전합니다.

01 — 배치

기기별 역할 — 어떤 계정을 어디에

가장 소중한 본계정은 한 대에만 두어 깨끗하게 지키고, 자동 준비작업이 많은 부계정 2개는 항상 켜두는 맥미니에 모읍니다. 단, 부계정 둘은 크롬 프로필을 다르게, 시간도 다르게 씁니다.

사장님 개인 기기 · 사람이 직접

💻 MacBook Air

본계정
  • 사장님이 평소 쓰는 기기 → 본계정만 여기에. 다른 계정 로그인 금지.
  • 후보 검토 + "보내기"는 여기서 사람이 직접 누름.
  • SOT R4 양보: 사장님이 크롬을 쓰기 시작하면 자동 작업 즉시 멈춤 → 어차피 발송은 사람이라 충돌 없음.
  • 가볍게: 좋아요·댓글 등 "사람다운 활동"의 주 무대.

상시 가동 운영기 · 준비작업 담당

🖥️ Mac mini

부계정 1부계정 2
  • 항상 켜둠 → 후보 수집·정제·줄 세우기(자동)를 밤낮으로.
  • 부계정 1 = 크롬 프로필 A, 부계정 2 = 크롬 프로필 B (또는 크롬/엣지로 아예 분리).
  • 두 부계정 동시 로그인 절대 금지 — 시간대를 나눠서(아래 03번).
  • 사장님이 거의 안 건드리는 기기라 R4 충돌이 적음 → 준비작업에 최적.
  • 여기서도 발송은 사람: 미니에 만들어진 "오늘 후보"를 사람이 검토·클릭.

왜 본계정을 미니에 안 두나? — 미니는 부계정 2개가 같이 사는 기기라 "여러 계정이 묶인 기기"입니다. 가장 소중한 본계정은 그 묶음에서 빼서 에어에 홀로 두는 게 안전합니다.

02 — 분리

계정마다 브라우저 완전 분리하기

이게 가장 효과 큰 방어입니다. 핵심은 "한 크롬 프로필 = 한 계정"이고, 쿠키·로그인 정보를 절대 섞지 않는 것입니다.

  1. 계정마다 크롬 프로필 따로 — 크롬 우측 상단 프로필 → "추가"로 부계정1용, 부계정2용을 따로 만듦. (각자 별도 쿠키·캐시)
  2. 더 강하게는 브라우저를 다르게 — 부계정1은 크롬, 부계정2는 엣지(또는 Chromium) 식으로. 지문이 더 갈라짐.
  3. 자동화 로그인 세션도 계정별로 — Playwright를 쓸 때 계정마다 user-data-dir(로그인 저장 폴더)를 따로. 한 폴더 공유 금지.
  4. 비밀번호·복구 이메일·전화도 계정마다 다르게 — 같은 복구 이메일을 세 계정에 쓰면 바로 묶임.
  5. 수상한 무료 VPN·프록시는 쓰지 말 것 — 오히려 더 의심받음. 깨끗한 집/사무실 IP가 더 신뢰도 높음.

가장 흔한 실수 — 한 크롬 창에서 로그아웃하고 다른 계정으로 로그인, 또 로그아웃하고 다른 계정… 이러면 쿠키가 섞여 한 기기에 세 계정이 다 묶입니다. 반드시 프로필(또는 브라우저)을 분리해서, 계정을 바꿀 때 로그아웃이 아니라 프로필을 바꾸는 방식으로.

03 — 시간표

하루 시간 나누기 — 동시에 안 켜기

특히 한 기기(미니)에 사는 부계정 2개는 겹치는 시간에 동시에 활동하면 묶입니다. 아래처럼 시간대를 갈라 둡니다. 본계정(에어)은 사장님 활동 시간에 맞춰 자유롭게.

시간대활동 계정기기하는 일
오전 9–12시부계정 1Mac mini · 프로필 A신청 검토·발송(사람), 활동
수시본계정MacBook Air사장님 직접 사용·발송 (R4 양보)
오후 2–6시부계정 2Mac mini · 프로필 B신청 검토·발송(사람), 활동

핵심 두 가지 — ① 미니에서 부계정 1·2가 같은 시간에 둘 다 신청 활동 하지 않게(오전/오후로 갈라). ② 밤에 자동으로 도는 건 "후보 수집"까지만이고, 신청 발송은 낮에 사람이. 자동 로그인 자체는 SOT 1번대로 계속 유지합니다(끊지 않음).

지켜지는 SOT — ① 3사 자동 로그인 유지(끊지 않음) ② 사장님 크롬 보이면 에어 자동작업 양보 ③ "보내기"는 세 계정 모두 사람이 ④ 보고는 쉬운 한국어로.

05 — 점검표

설정 한 번 / 매일 확인

처음 한 번 설정

  • 에어 = 본계정만, 미니 = 부계정 1·2로 정함
  • 미니에 크롬 프로필 A(부계정1) / B(부계정2) 따로 생성
  • 계정마다 복구 이메일·전화 다르게 되어 있는지 확인
  • Playwright 로그인 폴더(user-data-dir)를 계정별로 분리
  • 부계정 활동 시간대(오전/오후) 정해서 캘린더에 표시

매일

  • 부계정 1·2가 같은 시간에 동시 신청하지 않았는가
  • 계정 바꿀 때 로그아웃 말고 프로필 전환으로 했는가
  • 보내기는 사람이 직접 눌렀는가 (자동 발송 0건)
  • 사장님이 에어 크롬 쓰기 시작하면 자동작업 멈췄는가
