```
docs/ai-search/ai-search-reservoir-strategy-2026-06-12.html 의 저수지 모델을 harness 표준 루프로 단계적 구현한다. 실시간 서치는 끄지 않고 병행으로 띄워 같은 포지션에 두 방식 순도를 비교한다. 모두 tools/multi_position_sourcing/ 위에서 증분으로.

[워크트리 규칙] main은 읽기 전용. 한 단계 = 이슈 1개 = 워크트리 1개(worktrees/<name>, 브랜치 task/<name>) = 인수기준 1개. make task NAME=... 로 파고, RED 테스트→구현→verify→make ship(PR)→CI 초록→merge→worktree 정리까지 그 안에서 끝낸다. 메인에서 직접 소스 수정 금지.

[구현 형태] 엔진(매칭·스코어링·임베딩·큐·로그)은 tools/ 아래 코드로 구현하고 pytest로 검증한다. 스킬(skills/)은 "저수지 돌려/매칭해줘" 진입점과 사람 판단 단계(세그먼트·골드시드·경계선 검수)만 얇게 맡아 코드를 호출한다. 매칭·점수 로직을 스킬 텍스트로 구현하지 않는다(재현성 위반).

[머신·로그인 정책 — 사이트 전담 고정 폐기]
- 맥북·맥미니·맥에어 3대 모두 사람인·잡코리아에 로그인하고 Harvest 허용. 한 머신이 한 사이트 전담이 아니라, 셋 다 두 사이트를 돈다(부하만 분산).
- 가동 시작 우선순위는 맥미니부터.
- 맥북은 사장님 Chrome 점유가 감지되면 그 머신 무인 워커 즉시 정지(R4) — 비는 시간에만 무인 Harvest.
- 링크드인 RPS는 시간대 시프트: 낮~밤(사장님 재석)은 맥북프로에서 유인 사용, 사장님 퇴근 후에는 맥미니로 로그인 전환해 야간 가동. 동시에 두 머신이 같은 RPS 세션에 로그인하지 않는다(세션 충돌 방지).

[저수지 저장·품질 기준 — 어떤 레쥬메를 우수로 보나]
- 저장은 무조건(사람인/잡코리아 상세진입=차감0이므로 발견 즉시 전부 저장). 단 후보 품질 점수를 매겨 우수 인재를 상위로 정렬한다. scoring.py 를 다음 4기준으로 확장:
  1) 우수 대학교 — 학교 티어 가점.
  2) 직무 기술 우수 — JD 직무 키워드·기술스택 직결성 가점.
  3) 좋은 회사 재직자 — 회사 티어 가점.
  4) 이직 안정성 — 경력 이력에서 재직기간을 계산해, 1년 미만 재직 후 이직을 연속 2회 이상 가진 사람은 감점(2회부터 감점, 3회 이상은 더 크게). 데이터가 없으면 프로필 캡처를 경력 이력(회사·입퇴사 시점)까지 확장하고 그 픽스처+테스트를 같은 커밋에.

[관측가능성 규칙 — 모든 단계 공통, 절대 생략 금지]
- 5라인(Harvest→Index→Match→Calibrate→Send)의 모든 경계와 모든 fail-closed 분기에 구조화 로그 1줄(JSON):
  {ts, run_id, machine, segment_id, site, line, in_count, out_count, dropped_count, status(ok|fail|skip), fail_reason, latency_ms}
- logs/reservoir/<YYYY-MM-DD>.jsonl 에 append. "어느 머신·어느 라인·어느 세그먼트·어느 사이트에서 몇 건이 왜 빠졌나"를 로그만 보고 즉시 답할 수 있어야 한다.
- 실패는 조용히 넘기지 않는다(fail-closed). 빠진 건수와 이유를 반드시 로그로 남긴다. 이 로그 스키마 자체를 verify 계약 테스트로 검증(필드 누락 시 RED). 새 라인/사이트/머신을 추가하면 그 로그 단언+픽스처를 같은 커밋에(자기확장 규칙).

[단계 — 순서대로, 한 번에 하나]
1. 세그먼트 taxonomy 고정: ClickUp active 포지션을 4~6 세그먼트(IT·AI·데이터 / 마케팅·그로스 / 세일즈·BD / HR·재무·운영)로 매핑. grouping.py 규칙 키를 segment_id 로 승격, keywords.py·models.py(industry_segment→segment_id) 반영. RED: 포지션→세그먼트 매핑이 결정론적.
2. Harvest 큐 ↔ Match 큐 분리 + 3머신 멀티사이트 로그인: 포지션 트리거 없이 세그먼트만으로 도는 연속 Harvest 사이클 신설. queue_runner.py·portal_queue_executor.py·portal_worker.py. 위 머신·로그인 정책 반영. RED: Harvest 큐가 포지션 없이 돌고 발견 프로필을 무조건 저장 + 머신별 사이트 배정이 정책대로.
3. 저수지 품질 스코어링 확장: scoring.py 에 위 4기준(특히 이직안정성: 1년 미만 이직 연속 2회부터 감점, 3회 이상 더 크게) 추가. RED: 1년 미만 이직 2회/3회 프로필이 단계적으로 감점되고, 우수대학·좋은회사·직무직결이 가점되는 테스트.
4. pgvector 스키마 + 임베딩 적재: Supabase 프로필 임베딩 테이블/컬럼 + HNSW 인덱스. 아카이버 저장 직후 임베딩 훅. 신규 embeddings.sql·embed.py. RED: 저장→임베딩 적재 + 중복 dedup.
5. match() + 재랭킹: JD 임베딩 → 세그먼트 내 top-K 코사인 → scoring.py(4기준) 2차 정렬 → 알바용 정렬 줄. 신규 match.py. RED: 같은 JD·같은 저수지면 같은 순서(결정론적 재현성).
6. A/B 측정 하니스: 동일 포지션에 [실시간] vs [저수지] 후보를 나란히 산출, 헤드헌터 블라인드 채점. 순도 비교 리포트.

[SOT 불변식 — 절대 약화 금지] 3사 자동로그인 안 막음 · 맥북 Chrome 점유 감지 시 무인 워커 즉시 정지(R4) · Send 자동발송 금지(사람 게이트) · 사장님께 쉬운 한국어로 보고.

[완료 판정 숫자] 매칭 순도 top-20 중 적합도≥85 비율 ≥70% · 알바 발송 ≥20/시 · 저수지 세그먼트별 누적 우상향 · 동일 포지션 재매칭 편차 ≈0.

지금은 커밋 1(세그먼트 taxonomy)만 시작한다.
```

## 핵심 기능 요약

| # | 기능 | 한 줄 | 형태 |
|---|---|---|---|
| 1 | 직군 나누기(세그먼트) | 포지션을 4~6 직군으로 묶음 | 코드 |
| 2 | 계속 모으기(Harvest) | 포지션 없어도 24h 후보 수집·무조건 저장 | 코드 |
| 3 | 품질 점수 | 대학·직무·회사·이직안정성으로 우수 인재 상위 정렬 | 코드 |
| 4 | 임베딩 색인 | 후보를 숫자로 바꿔 pgvector에 저장 | 코드 |
| 5 | 즉시 매칭 | JD top-K 벡터검색 + scoring 재랭킹 | 코드 |
| 6 | A/B 측정 | 실시간 vs 저수지 블라인드 채점 | 코드 + 사람 |

- **머신:** 맥북·맥미니·맥에어 모두 사람인·잡코리아 로그인(전담 없음, 시작은 맥미니). 링크드인은 낮=맥북·밤=맥미니 시프트.
- **로그:** 모든 단계가 `logs/reservoir/날짜.jsonl`에 "몇 건 들어와 몇 건 나갔고 왜 빠졌나 + 어느 머신"을 남겨 고장 지점 즉시 추적.
- **구현 형태:** 엔진=코드(pytest 검증), 스킬=얇은 진입점·사람 판단만. 매칭/점수를 스킬 텍스트로 만들지 않음(재현성).
