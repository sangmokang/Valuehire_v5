## 절대 규칙 (사장님 명시 — 약화 금지)
- ❌ **전화통화 요청 금지** ("5분만 통화", "통화하자" 등).
- ❌ **"꼭 맞아서 / 정확히 맞물린다 / 딱 맞다"류 과장 금지.**
  (단, R21 표준 P.S. "지금 이 포지션이 **딱 맞지 않으셔도** 괜찮습니다"는 부정문 — 과장이 아니므로 허용.)
- ❌ **후보 이력을 요약·나열하지 않는다.** 뉘앙스는 "내가 당신 이력을 살펴보고 제안한다" + **정곡을 찌르는 관찰 1줄**만.
- ✅ 개인화 오프너 뒤에는 **회사 브리핑 + JD**(자격요건·주요업무)를 붙인다. 관련 JD를 ClickUp/공식 채용에서 찾아 **윤문**.
- ✅ **가독성**: 문단·불릿으로 나눈다(한 덩어리 텍스트 금지).
- ❌ raw `{{...}}`·중괄호 `{` `}`·HTML 주석(`<!--`) 금지 (jd-set-builder R25 — invalid variable 배너).

## 언어 자동 선택 (2026-07-03 신설)

- **한국어 프로필** → 본문 한국어.
- 판정은 기계로: `body_language_for_profile(name, visible_text)` (`tools/multi_position_sourcing/inmail_precheck.py`).

## 발송 전 기계 체크리스트 (2026-07-03 신설 — 통과 전 문구 제공 금지)

문구 완성 후 **반드시** 아래 CLI 를 돌린다. exit 0 이 아니면 STOP — 고쳐서 재검사.

```bash
python3 -m tools.multi_position_sourcing.inmail_precheck \
  --body-file /tmp/inmail-body.txt \
  --profile-name "<수확 results.json 의 name 그대로>" \
  --channel linkedin_rps \
  --briefing-elements <확인된 §1.5 요소 개수>
```

| # | 체크 | 판정 |
|---|------|------|
| ① | **인사말 이름 == 수확 프로필 이름** (부분 일치, 추출 실패도 STOP) | 다르면 **STOP** |
| ② | **글자수 한도** — LinkedIn InMail **1,899자** / 사람인·잡코리아 **2,000자** (NFC 문자수) | 초과 **STOP** |
| ③ | **금지 워딩 린트** — 통화/전화, 딱 맞·정확히 맞물·꼭 맞, `{{`/`}}`·중괄호, HTML 주석 | 검출 **STOP** |
| ④ | **회사 브리핑 요소 6개 미만** (§1.5 8요소 기준) | 사장님 **보고 후 진행** |
| ⑤ | **한글 자모 분리·기지 오타("하니다")** + 컴포저 입력 후 **스크린샷 대조** | 검출 **STOP** |
| ⑥ | **VERIFIED-PULL(무료 이력서 피드백) 문단 존재** | 부재 **STOP** |

- ①의 이름은 **수확 JSON 의 `name` 을 그대로** 인사말에 쓴다 — 손으로 옮겨 적지 않는다.
- ⑤의 기계 검사는 자모 분리·기지 오타까지만 잡는다 — 컴포저에 **입력한 뒤** 글자수·깨짐을
  **스크린샷으로 대조**하는 절차(SKILL "실행 함정" 절)는 그대로 필수.
- 반면교사 = **Movensys 사고(2026-06-30)**: 수신자 Meseret Abayebas Tadese 에게 "Rocha연구원님"
  인사(①) + "하니다" 오타(⑤) + VERIFIED-PULL·P.S. 누락(⑥)이 그대로 발송됨. 이 체크리스트가 셋 다 차단한다.
