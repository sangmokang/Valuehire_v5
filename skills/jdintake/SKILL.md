# jdintake — 웹 JD 수집

## 0. 이 스킬이 하는 일 / 안 하는 일

| | |
|---|---|
| **한다** | 회사의 **공식** 채용페이지를 찾고, 그 공고 원문에서 JD를 구조화해 수집·보고 |
| **안 한다** | 후보자 검색(`aisearch`), 제안 발송(금지), 채용공고 게시·4채널 등록(`jdbuilder`) |

호출 경로: 자연어 셸 SOT-32 라우팅표 `(web, find)` → 큐 스킬 `jdintake`.
정본 계약: `docs/sot/32-nl-shell-routing.json`.

## 1. 절대 금지 (이 스킬의 실패는 대부분 여기서 난다)

1. **JD 날조 금지.** 공고를 못 찾았으면 **못 찾았다고 보고**한다. 회사 소개나 다른 공고로 추측해 채우지 않는다. 확인 안 된 항목은 `미확인`으로 남긴다 — 빈 칸을 그럴듯한 문장으로 메우는 것이 이 스킬의 가장 큰 위험이다.
2. **발송 금지.** 수집 스킬이다. 어떤 경로로도 메일·InMail·제안을 **발송하지 않는다**(SOT-28, SOT-32 F-NL1).
3. **채용사이트 raw 조작 금지.** 사람인·잡코리아·LinkedIn 을 즉석 CDP·스크립트로 직접 조작하지 않는다(SOT-25 §0). 이 스킬은 공개 웹 페이지를 읽을 뿐이다.
4. **포지션 등록 금지.** 수집한 JD로 ClickUp·채용사이트에 등록하지 않는다 — 그건 `jdbuilder` 단일 진입점 소관(SOT-25).

## 2. 출처 우선순위 (E-NL3)

**공식 > 공식 대행 > 그 외.** 어떤 것을 썼든 **출처 URL 을 반드시 표기**한다.

| 순위 | 유형 | 예 |
|---|---|---|
| 1 | 회사 자체 채용페이지 | `<회사도메인>/careers`, `/recruit`, `/jobs` |
| 2 | 회사가 쓰는 공식 채용 솔루션 | greetinghr, Greenhouse, Lever, Ashby, 워크데이 |
| 3 | 공식 채용 플랫폼 공고 | 원티드, 잡플래닛 채용, 사람인·잡코리아 **공고**(인재검색 아님) |
| ✗ | 2차 가공·커뮤니티 | 블라인드, 잡플래닛 리뷰, 블로그, 뉴스 요약 |

4순위(✗)만 찾았다면 **JD 수집 실패로 보고**한다. 2차 가공물을 원문처럼 쓰지 않는다.

## 3. 절차

1. **회사 확정** — 동명 회사가 여럿이면 진행하지 말고 후보를 제시한다(SOT-32 §4 many_hits).
2. **공식 채용페이지 탐색** — 회사 도메인의 `/careers`·`/recruit`·`/jobs` 를 먼저 본다. 없으면 §2 순위대로 내려간다.
3. **직무 공고 특정** — 요청한 직무(역할 약어는 SOT-32 `role_synonyms` 로 확장)와 맞는 공고를 고른다. 여러 건이면 후보 제시 후 정지.
4. **원문 수집** — 공고 페이지 본문을 그대로 확보한다(요약본 아님).
5. **구조화** — §4 형식으로 정리한다.
6. **보고** — 출처 URL + 수집 시각을 함께 남긴다.

## 4. 산출 형식

```
## <회사> — <포지션명>
- 출처: <URL>  (유형: 공식 자체 / 공식 솔루션 / 플랫폼 공고)
- 수집 시각: <YYYY-MM-DD HH:MM KST>

### 담당 업무
### 자격 요건
### 우대 사항
### 고용형태·근무지·연봉
### 채용 절차
### 미확인 항목
- (원문에 없는 항목을 여기 나열 — 추측해서 위 칸을 채우지 않는다)
```

## 5. 정지 조건 (사람을 부른다)

- 로그인·캡차·봇차단이 걸린 페이지
- 회사가 특정되지 않음 / 공고 후보가 여럿
- §2 의 1~3순위 출처를 못 찾음 → **실패 보고**(2차 가공물로 대체하지 않음)
