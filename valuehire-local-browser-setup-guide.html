Valuehire 운영 가이드

로컬 브라우저 실행 환경 설정 가이드

이 문서는 디스코드 명령을 받은 뒤, 실제 브라우저 자동화 작업은 사용자의 맥에서 실행되도록 설정하는 방법을 안내합니다. 핵심은 Claude 쪽 확장 설정이 아니라 Hermes 게이트웨이, 로컬 브라우저 자동화, 자동화 전용 크롬 프로필을 안전하게 연결하는 것입니다.

대상 독자: Valuehire 운영자, 채용 운영 담당자, 로컬 실행 환경을 처음 세팅하는 사용자

1. 먼저 알아야 할 결론

결론: “MCP Chrome in Claude”는 필수가 아닙니다. 디스코드 명령을 로컬 맥에서 실행하려면 Hermes 게이트웨이와 로컬 브라우저 자동화가 핵심입니다.

쉽게 말하면, 식당 주문은 디스코드에서 받고 실제 요리는 사용자 맥에서 하는 구조입니다. 이때 필요한 것은 주문을 받는 Hermes 게이트웨이와, 요리를 맡는 로컬 크롬 자동화 환경입니다.

  • Hermes 직접 브라우저 방식: 가장 단순하고 우선 추천하는 방식입니다.
  • Hermes에 Playwright MCP 연결: 브라우저 조작 안정성이 더 필요할 때 선택합니다.
  • Claude Desktop 또는 Claude Code 중심 방식: 중간 단계가 늘어나므로 운영 파이프라인에는 덜 권장합니다.

2. 권장 실행 구조

권장 구조는 아래와 같습니다.

디스코드 사용자 명령
        ↓
Hermes 게이트웨이
        ↓
사용자 로컬 맥
        ↓
자동화 전용 크롬 프로필
        ↓
사람인 · 잡코리아 · 링크드인 검색 또는 확인

왜 이 구조가 좋은가요?

  • 실제 실행 위치를 특정 사용자의 맥으로 고정할 수 있습니다.
  • 사람인, 잡코리아, 링크드인 로그인 세션을 로컬에 보관할 수 있습니다.
  • 개인용 크롬과 자동화용 크롬을 분리할 수 있습니다.
  • 저장, 등록, 발송 같은 위험 작업을 승인 게이트로 막을 수 있습니다.

주의할 점

  • 로컬 맥이 켜져 있어야 합니다.
  • 인터넷 연결이 안정적이어야 합니다.
  • 자동화용 크롬 프로필에 각 사이트 로그인이 되어 있어야 합니다.
  • 링크드인은 자동화 탐지 위험이 높으므로 자동 발송하지 않습니다.

3. 준비물

아래 항목이 준비되어 있어야 합니다.

항목 용도 확인 명령
Hermes 디스코드 명령 수신과 로컬 작업 실행 hermes status --all
Hermes 게이트웨이 디스코드와 Hermes 연결 hermes gateway status
browser 도구 브라우저 자동화 hermes tools list
computer_use 도구 로컬 화면 조작 보조 hermes tools list
Google Chrome 자동화용 브라우저 open -a "Google Chrome"
Node.js, npx Playwright MCP를 쓸 때 필요 node -v && npx -v
uvx 파이썬 계열 실행 도구가 필요할 때 사용 uvx --version
참고: 도구를 새로 켠 뒤에는 현재 세션이 아니라 새 세션부터 적용될 수 있습니다. 게이트웨이에서는 재시작이 필요할 수 있습니다.

4. 설정 순서

실행 방식을 먼저 정합니다

처음에는 Hermes 직접 브라우저 방식을 추천합니다. 단순하고 운영 위험이 적습니다.

  • 기본 추천: Hermes 직접 브라우저 사용
  • 추가 안정화가 필요할 때: Hermes에 Playwright MCP 연결

Hermes 게이트웨이 상태를 확인합니다

hermes gateway status
hermes status --all

게이트웨이가 꺼져 있다면 아래 명령으로 시작합니다.

hermes gateway start

처음 설치하는 사용자라면 아래 순서로 설정합니다.

hermes setup
hermes gateway setup
hermes gateway install
hermes gateway start

브라우저 관련 도구를 켭니다

아래 명령으로 도구 상태를 확인합니다.

hermes tools list

필요하면 브라우저 도구를 켭니다.

hermes tools enable browser
hermes tools enable computer_use

설정 후에는 새 세션을 시작하거나 게이트웨이를 재시작합니다.

hermes gateway restart

자동화 전용 크롬 프로필을 만듭니다

개인용 크롬과 자동화용 크롬을 섞지 않는 것이 중요합니다. 아래 경로처럼 별도 프로필 디렉터리를 사용합니다.

mkdir -p ~/.cache/valuehire-recruiting-chrome

자동화 전용 프로필로 크롬을 실행합니다.

open -na "Google Chrome" --args \
  --user-data-dir="$HOME/.cache/valuehire-recruiting-chrome" \
  --profile-directory="Default"
이 창은 일반 크롬과 별개로 로그인 상태를 저장합니다. 앞으로 채용 자동화 테스트는 이 프로필에서만 진행합니다.

채용 사이트 로그인을 수동으로 완료합니다

자동화 전용 크롬 창에서 아래 사이트에 직접 로그인합니다.

  • 사람인 기업 계정
  • 잡코리아 기업 계정
  • 링크드인 계정
로그인, 보안 인증, 캡차는 사람이 직접 처리해야 합니다. 자동화로 우회하려고 하면 계정 제한 위험이 커집니다.

디스코드 명령 라우팅을 확인합니다

디스코드에서 “후보자 서치”, “포지션 검색”, “사람인 잡코리아 링크드인 확인” 같은 명령이 Valuehire 도구로 연결되는지 확인합니다.

처음에는 반드시 읽기 전용 또는 드라이런으로만 실행합니다.

예시 명령:
후보자 서치 드라이런으로 확인해줘
사람인 잡코리아 링크드인 로그인 상태만 점검해줘
3채널 채용 파이프라인 dry-run으로 확인해줘

위험 작업은 승인 게이트 뒤에 둡니다

다음 작업은 명시 승인 없이는 실행하지 않도록 운영합니다.

  • 채용공고 등록
  • 후보자 저장
  • 후보자 제안 발송
  • 링크드인 메시지 발송
링크드인은 자동 발송하지 않는 것이 원칙입니다. 결과는 “사람 확인 필요” 상태로 두고, 담당자가 직접 복사해 발송합니다.

5. 선택 사항: Playwright MCP 연결

MCP는 모델 컨텍스트 프로토콜의 줄임말입니다. 쉽게 말하면 Hermes가 외부 도구 서버와 대화할 수 있게 해주는 연결 규격입니다. Playwright MCP는 브라우저 자동화를 더 안정적으로 다루기 위한 선택지입니다.

먼저 현재 MCP 설정을 확인합니다

hermes mcp list

No MCP servers configured.라고 나오면 아직 MCP 서버가 등록되지 않은 상태입니다.

Playwright MCP를 등록할 때의 예시

아래 명령은 환경에 따라 패키지명이나 옵션이 달라질 수 있습니다. 조직 표준 명령이 있다면 그 명령을 우선 사용합니다. 
hermes mcp add playwright --command "npx @playwright/mcp@latest"
hermes mcp test playwright
hermes mcp configure playwright

언제 MCP를 붙이면 좋나요?

  • Hermes 기본 브라우저 도구만으로는 특정 사이트 조작이 불안정할 때
  • 브라우저 자동화 도구를 표준화해서 여러 사용자가 같은 방식으로 쓰고 싶을 때
  • 브라우저 세션과 자동화 명령을 더 명확히 분리하고 싶을 때

언제 굳이 안 붙여도 되나요?

  • 현재 Hermes browser 도구로 충분히 드라이런이 되는 경우
  • 단순 검색, 화면 확인, 로그인 상태 점검 정도만 필요한 경우
  • 운영 파이프라인을 최대한 단순하게 유지하고 싶은 경우

6. 안전 게이트 원칙

채용 운영 자동화는 작은 클릭 하나가 실제 고객, 후보자, 채용 플랫폼에 영향을 줄 수 있습니다. 그래서 기본값은 항상 닫아두는 것이 안전합니다.

작업 기본 상태 운영 원칙
검색 허용 가능 읽기 전용으로 시작합니다.
후보자 저장 차단 명시 승인 후에만 저장합니다.
채용공고 등록 차단 공고 내용과 대상 플랫폼 확인 후 등록합니다.
사람인·잡코리아 제안 발송 차단 담당자 승인 후에만 발송합니다.
링크드인 메시지 발송 자동 발송 금지 초안까지만 만들고 사람이 직접 발송합니다.
운영 원칙은 “검색은 자동화할 수 있지만, 외부에 남는 행동은 사람이 승인한다”입니다.

7. 드라이런 테스트

처음에는 결과 요약까지만 확인합니다. 저장, 등록, 발송은 모두 꺼둡니다.

  1. 자동화 전용 크롬 프로필을 실행합니다.
  2. 사람인, 잡코리아, 링크드인 로그인 상태를 확인합니다.
  3. 디스코드에서 드라이런 명령을 보냅니다.
  4. Hermes가 어떤 단계까지 접근했는지 로그를 확인합니다.
  5. 후보 검색 결과 요약만 받습니다.

권장 테스트 문장

3채널 채용 파이프라인을 드라이런으로만 점검해줘.
저장, 등록, 제안 발송은 하지 말고 로그인 상태와 후보 검색 가능 여부만 알려줘.

게이트웨이 로그 확인

tail -n 80 ~/.hermes/logs/gateway.log

8. 문제 해결

디스코드 명령에 반응이 없습니다

  • hermes gateway status로 게이트웨이가 켜져 있는지 확인합니다.
  • 디스코드 봇의 메시지 내용 읽기 권한이 켜져 있는지 확인합니다.
  • 게이트웨이를 재시작합니다: hermes gateway restart

브라우저 도구가 보이지 않습니다

  • hermes tools list로 browser와 computer_use 상태를 확인합니다.
  • 필요하면 hermes tools enable browser로 켭니다.
  • 도구 변경 후 새 세션을 시작하거나 게이트웨이를 재시작합니다.

로그인이 자꾸 풀립니다

  • 자동화 전용 크롬 프로필 경로가 매번 같은지 확인합니다.
  • 사이트 보안 정책상 재로그인이 필요할 수 있습니다.
  • 캡차나 2단계 인증은 직접 처리합니다.

Playwright MCP가 연결되지 않습니다

  • node -v와 npx -v가 정상인지 확인합니다.
  • hermes mcp list로 등록 여부를 확인합니다.
  • hermes mcp test playwright로 연결 테스트를 합니다.

외부 작업이 실행될까 봐 불안합니다

  • 명령에 “드라이런”, “읽기 전용”, “저장하지 마”, “발송하지 마”를 명확히 넣습니다.
  • 운영자는 저장, 등록, 발송 게이트가 닫혀 있는지 확인합니다.
  • 링크드인은 항상 사람이 직접 확인하고 발송합니다.

9. 최종 체크리스트

여기까지 끝나면 “디스코드에서 명령을 받고, 로컬 맥의 분리된 크롬 프로필에서 안전하게 검색을 수행하는 기본 환경”이 준비된 것입니다.
